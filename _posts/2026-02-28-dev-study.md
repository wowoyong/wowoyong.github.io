## 핵심 요약

데이터베이스는 데이터를 체계적으로 저장하고 관리하는 시스템으로, 모든 현대 소프트웨어 시스템의 근간을 이룹니다. 학부 과정에서는 데이터의 효율적인 저장, 검색, 관리를 위한 기본 개념과 기술을 학습합니다. 관계형 데이터베이스의 이론과 구조적 질의어를 통한 데이터 조작 방법을 중심으로, 데이터 모델링 및 정규화 과정을 깊이 있게 다룹니다. 또한, 처리 단위 관리와 동시성 제어 같은 고급 주제를 통해 데이터 무결성과 일관성을 확보하는 방법을 배웁니다. 최근에는 대용량 데이터와 클라우드 컴퓨팅 환경에 맞춰 비관계형 데이터베이스와 분산 데이터베이스 기술의 중요성도 커지고 있습니다.

## 데이터베이스의 이해

- 데이터베이스는 데이터를 체계적으로 저장, 관리, 검색하는 시스템입니다.
- 데이터 중복을 최소화하고 데이터 일관성을 유지하는 것이 중요합니다.
- 데이터베이스 관리 시스템은 데이터베이스를 생성, 유지보수하는 소프트웨어입니다.
- 사용자와 응용 프로그램이 데이터를 효율적으로 접근할 수 있도록 돕습니다.
- 데이터베이스 시스템의 3단계 구조: 외부 구조, 개념 구조, 내부 구조를 이해합니다.
- 데이터 독립성(논리적, 물리적) 개념을 통해 유연한 시스템 구축이 가능합니다.
- 다양한 데이터 모델 중 관계형 모델이 가장 널리 사용됩니다.
- 정보의 영속성을 보장하여 시스템 재시작 후에도 데이터를 유지합니다.

## 관계형 데이터베이스

- 데이터를 행과 열로 구성된 표 형태로 저장하는 모델입니다.
- 각 표는 고유한 기본 키를 가지며, 이는 행을 식별합니다.
- 외래 키를 사용하여 표 간의 관계를 정의하고 참조 무결성을 유지합니다.
- 관계, 행, 속성과 같은 용어를 학습합니다.
- 관계형 대수와 관계형 해석은 이론적 기반입니다.
- 정규화는 데이터 중복을 제거하고 이상 현상을 방지하는 과정입니다.
- 1차 정규형부터 3차 정규형, 보이스-코드 정규형까지 단계별 정규화 원칙을 이해합니다.
- 개체-관계 다이어그램은 데이터베이스 설계를 시각화하는 도구입니다.

## 구조적 질의어

- 관계형 데이터베이스의 데이터를 조작하고 정의하는 표준 언어입니다.
- 데이터 정의어: `CREATE`, `ALTER`, `DROP` 등으로 구조를 정의합니다.
- 데이터 조작어: `SELECT`, `INSERT`, `UPDATE`, `DELETE` 등으로 데이터를 다룹니다.
- 데이터 제어어: `GRANT`, `REVOKE` 등으로 접근 권한을 관리합니다.
- `SELECT` 문은 `WHERE`, `GROUP BY`, `HAVING`, `ORDER BY` 절과 함께 복잡한 질의를 작성할 수 있습니다.
- 결합 연산을 통해 여러 표의 데이터를 결합하여 가져올 수 있습니다.
- 하위 질의와 보기를 활용하여 질의를 효율적으로 구성합니다.
- 색인은 데이터 검색 속도를 향상시키기 위한 핵심 요소입니다.

## 데이터베이스 설계

- 사용자의 요구사항을 분석하여 효율적인 데이터 모델을 구축하는 과정입니다.
- 개념적 설계, 논리적 설계, 물리적 설계의 세 단계를 거쳐 진행됩니다.
- 개체-관계 모델링을 통해 개체, 속성, 관계를 식별하고 표현합니다.
- 정규화 과정을 통해 데이터 중복을 최소화하고 데이터 무결성을 확보합니다.
- 비정규화는 성능 최적화를 위해 의도적으로 중복을 허용하는 전략입니다.
- 적절한 색인을 설계하여 질의 성능을 최적화하고 응답 시간을 단축합니다.
- 자료형 선택, 제약 조건 설정 등 물리적 저장 방식을 고려합니다.
- 보안, 자료 보존 및 복구 전략도 초기 설계 단계에서 함께 고려되어야 합니다.

## 처리 단위와 동시성 제어

- 처리 단위는 데이터베이스의 논리적 작업 단위를 의미하며, 모든 작업을 한 번에 수행하거나 모두 취소합니다.
- 원자성, 일관성, 고립성, 지속성의 속성을 만족해야 합니다.
- 동시성 제어는 여러 처리 단위가 동시에 실행될 때 데이터 일관성을 유지하는 기법입니다.
- 잠금 기반 규약(예: 2단계 잠금 규약)이 가장 일반적인 동시성 제어 방식입니다.
- 시간 기록 기반 규약은 처리 단위 실행 순서를 미리 결정하여 충돌을 방지합니다.
- 교착 상태 발생 가능성과 이를 해결하는 기법(탐지 및 복구, 회피, 예방)을 학습합니다.
- 복구 관리자는 시스템 장애 발생 시 데이터베이스를 일관된 상태로 되돌립니다.
- 기록 파일은 처리 단위의 변경 내역을 기록하여 복구 작업에 활용됩니다.

## 비관계형 데이터베이스

- 관계형 데이터베이스의 한계를 극복하기 위해 등장한 비관계형 데이터베이스입니다.
- 대용량 데이터, 실시간 웹 응용 프로그램 등 유연성과 확장성이 중요한 환경에 적합합니다.
- 주로 분산 환경에서 수평적 확장을 통해 대용량 데이터를 처리합니다.
- 일관성보다는 가용성과 분할 허용성을 중시하는 원칙을 따릅니다.
- 주요 유형으로는 키-값 저장소, 문서형, 열 지향형, 그래프 데이터베이스 등이 있습니다.
- 몽고디비, 카산드라, 레디스, 네오포제이 등이 대표적인 비관계형 데이터베이스입니다.
- 구조 없이 데이터를 저장할 수 있어 데이터 모델 변경에 유연하게 대응합니다.
- 관계형 모델에서 복잡한 결합이 필요한 경우 비관계형 데이터베이스에서는 비정규화된 형태로 저장하여 성능을 높입니다.

## 최신 흐름

- 클라우드 컴퓨팅 데이터베이스의 확산: AWS RDS, Google Cloud SQL, Azure SQL Database 등 서비스형 데이터베이스가 보편화되고 있습니다.
- 데이터 호수 및 데이터 창고의 통합 및 발전: 정형/비정형 데이터를 아우르는 통합 분석 환경이 중요해지고 있습니다.
- 인메모리 데이터베이스의 성능 향상: 데이터를 주 메모리에 저장하여 극도로 빠른 처리 속도를 제공, 실시간 분석 및 거래 시스템에 활용됩니다.
- 분산 데이터베이스 및 마이크로서비스 설계 지원: 서비스별 독립적인 데이터베이스를 사용하는 패턴이 증가하며 분산 처리 단위 관리가 중요해집니다.
- 그래프 데이터베이스의 성장: 복잡한 관계형 데이터를 효율적으로 모델링하고 분석하여 사회 관계망, 추천 시스템 등에서 활용이 증대됩니다.
- 다중 모델 데이터베이스의 등장: 여러 데이터 모델(관계형, 문서형, 그래프 등)을 하나의 시스템에서 지원하여 유연성을 높입니다.
- 데이터 관리 체계 및 규제 준수 강화: 일반 데이터 보호 규정 등 개인정보보호법 준수를 위한 데이터 관리 및 보안 강화가 필수적입니다.
- 데이터베이스 보안의 중요성 증대: 암호화, 접근 제어, 감사 기록 등 데이터 유출 방지를 위한 기술 및 정책 도입이 강조됩니다.
- 기계 학습 기반의 데이터베이스 자동화 및 최적화: 자율 운영 데이터베이스는 관리자의 개입 없이 자체적으로 성능을 최적화하고 보안을 강화합니다.
- 분산 원장 기술과 데이터베이스의 결합 연구: 분산 원장 기술을 활용하여 데이터의 투명성, 불변성, 무결성을 보장하는 새로운 데이터 저장 방식이 탐구됩니다.

## 데이터베이스 관리 및 최적화

- 데이터베이스의 성능을 지속적으로 감시하고 조정하는 과정이 중요합니다.
- 질의 최적화는 느린 질의를 식별하고 실행 계획을 분석하여 성능을 개선합니다.
- 적절한 색인 생성 및 관리는 검색 속도를 향상시키고 쓰기 성능 저하를 방지합니다.
- 저장 절차, 함수, 자동 실행 등 데이터베이스 객체를 활용하여 업무 논리를 구현합니다.
- 데이터베이스 자료 보존 및 복구 전략을 수립하여 재해 발생 시 데이터 손실을 최소화합니다.
- 사용자 권한 관리 및 보안 설정은 데이터에 대한 무단 접근을 방지하는 핵심 요소입니다.
- 데이터 압축, 분할 등 저장 공간 효율화 기법을 적용할 수 있습니다.
- 데이터베이스 묶음 구성 및 복제를 통해 고가용성과 확장성을 확보합니다.
