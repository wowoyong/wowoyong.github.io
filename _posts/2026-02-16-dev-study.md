## 핵심 요약

프로그래밍은 컴퓨터에게 특정 작업을 수행하도록 지시하는 명령어 집합을 작성하는 과정입니다. 이는 소프트웨어 개발의 핵심 역량으로, 문제 해결 능력을 기르고 논리적 사고를 훈련하는 데 필수적입니다. 학부 과정에서는 다양한 프로그래밍 언어를 학습하며, 변수, 조건문, 반복문, 함수 등 기본 개념을 익힙니다. 또한, 코드 가독성과 효율성을 높이는 방법, 오류를 디버깅하는 기술도 함께 배웁니다. 프로그래밍은 단순한 코딩을 넘어 아이디어를 현실화하는 창조적인 활동입니다.

## 프로그래밍 개요

- 컴퓨터가 특정 작업을 수행하도록 명령어 집합을 작성하는 과정입니다.
- 문제 해결을 위한 논리적 사고와 절차를 컴퓨터가 이해할 수 있는 형태로 표현합니다.
- 소프트웨어, 응용 프로그램, 웹 서비스 등 다양한 디지털 제품의 근간을 이룹니다.
- 인간의 의도를 기계어로 번역하기 위해 프로그래밍 언어를 사용합니다.
- 명령어의 순서, 조건, 반복 등을 통해 복잡한 작업을 자동화할 수 있습니다.
- 창의적인 아이디어를 구체적인 기능으로 구현하는 핵심적인 기술입니다.
- 오류를 찾아 수정하는 디버깅 과정 또한 프로그래밍의 중요한 일부입니다.

## 프로그래밍 기본 요소

- **변수**: 데이터를 저장하고 참조하는 메모리 공간의 이름입니다.
- **자료형**: 변수에 저장될 수 있는 데이터의 종류(예: 정수, 실수, 문자열)를 정의합니다.
- **연산자**: 산술, 비교, 논리 등 데이터에 대한 다양한 작업을 수행하는 기호입니다.
- **조건문**: 특정 조건이 참 또는 거짓일 때 다른 코드 블록을 실행하도록 제어합니다 (예: '만약-그렇다면' 구문).
- **반복문**: 특정 코드 블록을 여러 번 반복하여 실행하도록 제어합니다 (예: '동안', '각각에 대해' 구문).
- **배열/리스트**: 동일한 자료형 또는 이종 자료형의 데이터를 순서대로 저장하는 자료 구조입니다.
- **주석**: 코드의 설명을 달아 가독성을 높이며, 프로그램 실행에는 영향을 미치지 않습니다.
- **입출력**: 프로그램이 외부로부터 데이터를 받거나 결과를 외부에 출력하는 기능입니다.

## 함수와 프로그램 구조

- **함수**: 특정 작업을 수행하는 코드 블록을 묶어 이름을 부여한 단위입니다.
- 코드의 중복을 줄이고 재사용성을 높여 프로그램 유지보수를 용이하게 합니다.
- 큰 프로그램을 작은 단위로 나누어 개발함으로써 복잡도를 관리합니다.
- 입력 값(매개변수)을 받아 처리하고, 결과 값(반환 값)을 돌려줄 수 있습니다.
- 모듈화된 함수들을 조합하여 더 크고 복잡한 기능을 구현할 수 있습니다.
- 지역 변수와 전역 변수의 개념을 이해하여 데이터 범위 관리가 중요합니다.
- 함수 호출 스택의 원리를 이해하는 것은 프로그램 실행 흐름 파악에 필수적입니다.
- 라이브러리나 프레임워크는 미리 작성된 함수와 모듈의 집합으로 개발 생산성을 높입니다.

## 객체 지향 프로그래밍

- 현실 세계의 객체를 모델링하여 프로그램을 설계하는 패러다임입니다.
- 데이터(속성)와 그 데이터를 조작하는 메서드(행동)를 하나의 객체로 묶습니다.
- **클래스**: 객체를 생성하기 위한 틀 또는 설계도이며, 객체의 속성과 행동을 정의합니다.
- **객체**: 클래스의 인스턴스로, 실제 메모리에 생성된 구체적인 개체입니다.
- **캡슐화**: 데이터와 메서드를 하나로 묶고 외부에서 직접 접근하는 것을 제한하여 데이터 무결성을 보장합니다.
- **상속**: 부모 클래스의 속성과 메서드를 자식 클래스가 물려받아 재사용성과 확장성을 높입니다.
- **다형성**: 같은 이름의 메서드가 객체의 타입에 따라 다른 동작을 수행하게 하는 능력입니다.
- **추상화**: 복잡한 구현 세부 사항을 숨기고, 사용자가 필요로 하는 핵심적인 기능만을 노출합니다.

## 프로그래밍 언어의 이해

- **파이썬**: 간결한 문법과 강력한 라이브러리로 데이터 과학, 웹 개발, 인공지능 분야에서 널리 사용됩니다.
- **자바**: 플랫폼 독립적인 특성으로 기업용 응용 프로그램, 안드로이드 모바일 개발에 강점을 가집니다.
- **C/C++**: 시스템 프로그래밍, 임베디드 시스템, 고성능 컴퓨팅 등 하드웨어 제어에 필수적입니다.
- **자바스크립트**: 웹 브라우저에서 동작하는 유일한 언어로, 웹 프론트엔드 및 백엔드 개발에 활용됩니다.
- **고 (Go)**: 구글이 개발한 언어로, 동시성 프로그래밍과 네트워크 서비스 구축에 효율적입니다.
- **러스트 (Rust)**: 높은 성능과 메모리 안전성을 동시에 제공하여 시스템 프로그래밍 분야에서 주목받고 있습니다.
- 각 언어는 특정 목적과 환경에 최적화된 특징을 가지므로, 프로젝트에 맞는 선택이 중요합니다.
- 여러 언어를 학습하면 문제 해결 관점을 넓히고 다양한 환경에 적응하는 능력을 기를 수 있습니다.

## 개발 환경 및 디버깅

- **통합 개발 환경**: 코드 편집, 컴파일, 디버깅 등 개발에 필요한 모든 기능을 한곳에 모아 제공합니다.
- 주요 통합 개발 환경으로는 비주얼 스튜디오 코드, 인텔리제이 아이디어, 이클립스 등이 있습니다.
- **컴파일러/인터프리터**: 작성된 소스 코드를 컴퓨터가 이해할 수 있는 기계어로 번역합니다.
- **버전 관리 시스템**: 코드 변경 이력을 추적하고 여러 개발자의 협업을 돕습니다 (예: 깃).
- **디버깅**: 프로그램 실행 중 발생하는 오류(버그)를 찾아내고 수정하는 과정입니다.
- **브레이크포인트**: 코드 실행을 특정 지점에서 멈춰 변수 값이나 실행 흐름을 검사하는 기능입니다.
- 오류 메시지를 읽고 이해하는 능력은 효율적인 디버깅의 핵심입니다.
- 단위 테스트를 작성하여 코드의 특정 부분이 예상대로 작동하는지 검증하는 습관이 중요합니다.

## 최신 흐름

- **인공지능 기반 코딩 지원 도구**: 깃허브 코파일럿, 챗지피티 등 인공지능이 코드 생성 및 오류 수정에 도움을 줍니다.
- **로우코드/노코드 플랫폼**: 코딩 지식 없이도 드래그 앤 드롭 방식으로 응용 프로그램을 개발할 수 있게 합니다.
- **웹 어셈블리**: 웹 브라우저에서 고성능 코드를 실행 가능하게 하여 웹 응용 프로그램의 가능성을 확장합니다.
- **양자 프로그래밍**: 양자 컴퓨터를 제어하는 알고리즘을 설계하며, 복잡한 문제 해결에 새로운 접근법을 제시합니다.
- **클라우드 네이티브 개발**: 컨테이너(예: 도커), 오케스트레이션(예: 쿠버네티스)을 활용하여 클라우드 환경에 최적화된 응용 프로그램을 만듭니다.
- **서버리스 컴퓨팅**: 개발자가 서버 관리에 신경 쓰지 않고 코드 실행에만 집중할 수 있도록 하는 모델입니다.
- **보안 코딩**: 처음부터 보안 취약점을 고려하여 안전한 코드를 작성하는 중요성이 더욱 강조되고 있습니다.
- **블록체인 기술과 스마트 계약**: 분산 원장 기술 기반의 응용 프로그램을 개발하는 프로그래밍 기법이 주목받고 있습니다.
- **지속적 통합/지속적 배포**: 개발, 테스트, 배포 과정을 자동화하여 소프트웨어 출시 주기를 단축합니다.
- **환경 친화적 프로그래밍**: 에너지 효율적인 코드 작성 및 자원 소비 최소화에 대한 관심이 증가하고 있습니다.
