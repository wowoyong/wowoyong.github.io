## 핵심 요약

컴퓨터 네트워크는 여러 컴퓨터와 기기들이 데이터를 주고받도록 연결된 시스템입니다. 인터넷의 근간을 이루며, 전 세계를 하나로 묶는 핵심 기술입니다. 학부 과정에서는 OSI 7 계층, TCP/IP 모델과 같은 기본 구조부터 IP 주소 체계, 주요 통신 프로토콜, 그리고 네트워크 보안 기술까지 폭넓게 학습합니다. 이 지식은 데이터 통신과 분산 시스템을 이해하는 데 필수적이며, 현대 소프트웨어 개발에서 매우 중요합니다. 특히, 클라우드 환경과 사물 인터넷 시대에 네트워크 지식은 더욱 중요해지고 있습니다. 안정적이고 효율적인 네트워크 설계 및 관리가 미래 기술 발전의 핵심이 될 것입니다.

## 네트워크 기초 및 구성 요소

- 네트워크 정의: 두 대 이상의 컴퓨터나 장치가 연결되어 자원을 공유하고 데이터를 교환하는 시스템입니다.
- 주요 목표: 자원 공유, 고속 통신, 신뢰성 향상, 확장성 확보 등이 있습니다.
- 네트워크 종류: 근거리 통신망, 광대역 통신망, 개인 영역 통신망 등으로 분류됩니다.
- 네트워크 장비: 라우터, 스위치, 허브, 브리지, 방화벽 등이 핵심 구성 요소입니다.
- 네트워크 토폴로지: 버스형, 링형, 스타형, 메시형 등 장치 연결 방식에 따른 구조입니다.
- 클라이언트-서버 모델: 자원을 요청하는 클라이언트와 자원을 제공하는 서버 간의 상호작용 방식입니다.
- 동등 계층 모델: 중앙 서버 없이 각 장치가 클라이언트이자 서버 역할을 동시에 수행하는 방식입니다.

## 네트워크 계층 모델

- OSI 7 계층: 네트워크 통신 과정을 7개의 추상적인 계층으로 나눈 표준 참조 모델입니다.
- 물리 계층(1): 비트 스트림의 물리적 전송을 담당하며, 케이블, 무선 신호 등이 여기에 해당합니다.
- 데이터 링크 계층(2): 인접한 두 장치 간의 데이터 전송을 관리하며, MAC 주소를 사용합니다.
- 네트워크 계층(3): 여러 네트워크를 통한 패킷의 최적 경로 선택 및 전달(라우팅)을 담당합니다.
- 전송 계층(4): 종단 간 데이터 전송의 신뢰성과 흐름 제어를 제공합니다.
- 응용 계층(7): 사용자에게 직접 서비스를 제공하며, HTTP, FTP, SMTP 등이 이 계층의 프로토콜입니다.
- TCP/IP 4/5 계층 모델: 인터넷 표준으로, OSI 모델을 간소화한 네트워크 모델이며, 실제 구현에 더 가깝습니다.

## 데이터 전송 기술 및 라우팅

- 패킷 교환: 데이터를 작은 조각(패킷)으로 나누어 전송하고 목적지에서 재조립하는 방식입니다.
- 회선 교환: 통신이 시작되기 전에 전용 통신 경로를 설정하여 데이터를 전송하는 방식입니다.
- 스위칭: 근거리 통신망 내에서 패킷을 효율적으로 전달하기 위해 MAC 주소를 기반으로 합니다.
- 라우팅: 최적의 경로를 찾아 패킷을 목적지 네트워크로 전달하며, IP 주소를 기반으로 합니다.
- 라우팅 프로토콜: RIP, OSPF, BGP 등은 라우터들이 경로 정보를 교환하고 최적 경로를 결정하는 데 사용됩니다.
- 유니캐스트, 멀티캐스트, 브로드캐스트: 데이터 전송 방식에 따른 목적지 지정 방법입니다.
- 흐름 제어: 송신자가 수신자의 처리 속도보다 빠르게 데이터를 보내는 것을 방지하는 기술입니다.
- 오류 제어: 전송 중 발생할 수 있는 데이터 오류를 탐지하고 수정하는 메커니즘입니다.

## IP 주소 체계와 도메인

- IP 주소: 네트워크에 연결된 각 장치를 식별하기 위한 고유한 논리적 주소입니다.
- IPv4: 32비트 주소 체계로, 네 개의 옥텟(예: 192.168.1.1)으로 표현되며 고갈 문제가 있습니다.
- IPv6: 128비트 주소 체계로, 훨씬 더 많은 주소를 제공하며 IPv4의 한계를 극복합니다.
- 서브넷 마스크: IP 주소에서 네트워크 부분과 호스트 부분을 구분하는 데 사용됩니다.
- 네트워크 주소 변환: 사설 IP 주소를 공인 IP 주소로 변환하여 인터넷에 접속하게 하는 기술입니다.
- 도메인 이름 시스템: 사람이 읽기 쉬운 도메인 이름(예: example.com)을 IP 주소로 변환해 줍니다.
- 동적 호스트 설정 프로토콜: 네트워크 장치에 IP 주소와 기타 네트워크 설정을 자동으로 할당합니다.
- 주소 결정 프로토콜: IP 주소를 물리적 MAC 주소로 변환하여 로컬 네트워크 내 통신을 가능하게 합니다.

## 주요 네트워크 프로토콜

- 전송 제어 프로토콜: 신뢰할 수 있는 연결 지향형 통신을 제공하며, 3-way 핸드셰이크를 사용합니다.
- 사용자 데이터그램 프로토콜: 비연결형 통신으로, 빠르지만 신뢰성을 보장하지 않습니다 (예: 스트리밍).
- 하이퍼텍스트 전송 프로토콜: 웹 브라우저와 웹 서버 간의 통신에 사용되는 기본 프로토콜입니다.
- 보안 하이퍼텍스트 전송 프로토콜: 하이퍼텍스트 전송 프로토콜에 SSL/TLS 암호화를 추가하여 보안을 강화한 버전입니다.
- 파일 전송 프로토콜: 서버와 클라이언트 간에 파일을 전송하는 데 사용되는 프로토콜입니다.
- 보안 셸: 원격 컴퓨터에 안전하게 접속하여 명령을 실행하거나 파일을 전송하는 데 사용됩니다.
- 단순 우편 전송 프로토콜, 우체국 프로토콜 버전 3, 인터넷 메시지 접근 프로토콜은 이메일 전송 및 수신/관리에 사용되는 프로토콜입니다.
- 텔넷: 원격 접속 프로토콜이지만 보안에 취약하여 요즘에는 보안 셸이 더 많이 사용됩니다.

## 네트워크 보안

- 방화벽: 네트워크 경계에서 내부 네트워크를 보호하고 불법적인 접근을 차단하는 시스템입니다.
- 가상 사설망: 공중망을 통해 안전하고 암호화된 전용 통신 채널을 구축하는 기술입니다.
- 침입 탐지 시스템/침입 방지 시스템: 네트워크 트래픽을 모니터링하여 비정상적인 활동을 탐지/차단합니다.
- 암호화: 데이터를 알아볼 수 없게 변환하여 정보 유출 시에도 내용을 보호하는 기술입니다.
- 인증 및 인가: 사용자의 신원을 확인하고(인증) 특정 자원에 대한 접근 권한을 부여(인가)하는 과정입니다.
- 분산 서비스 거부 공격: 여러 대의 컴퓨터로 특정 서버에 대량의 트래픽을 보내 서비스 마비를 유도합니다.
- 멀웨어: 컴퓨터 시스템을 손상시키거나 정보를 탈취하는 악성 소프트웨어(바이러스, 웜, 트로이 목마 등)입니다.
- 보안 프로토콜: 보안 소켓 계층/전송 계층 보안, 인터넷 프로토콜 보안 등은 데이터 통신의 기밀성, 무결성, 인증을 제공합니다.

## 최신 흐름

- 소프트웨어 정의 네트워킹: 네트워크 제어부를 데이터 전달부에서 분리하여 중앙에서 네트워크를 프로그래밍 방식으로 관리합니다.
- 네트워크 기능 가상화: 라우터, 방화벽 등 네트워크 기능을 전용 하드웨어 대신 범용 서버에서 소프트웨어로 구현합니다.
- 5G 이동 통신: 초고속, 초저지연, 초연결성을 제공하여 사물 인터넷, 자율주행, 실감형 콘텐츠의 기반을 마련합니다.
- 클라우드 네트워킹: 클라우드 환경에서 가상 네트워크를 구축하고 관리하며, 유연한 자원 확장을 지원합니다.
- 엣지 컴퓨팅: 데이터를 생성하는 장치나 그 근처에서 데이터를 처리하여 지연 시간을 줄이고 대역폭 사용을 최적화합니다.
- 사물 인터넷 네트워킹: 수많은 사물 인터넷 장치들의 효율적인 연결과 데이터 교환을 위한 특화된 통신 기술 및 프로토콜이 중요해집니다.
- 양자 네트워킹: 양자역학 원리를 활용하여 정보 전송의 절대적인 보안을 목표로 하는 차세대 통신 기술 개발이 진행 중입니다.
- 제로 트러스트 보안: '절대 믿지 말고 항상 검증하라'는 원칙에 따라 모든 사용자 및 장치에 대한 지속적인 인증과 권한 부여를 요구합니다.
- 인공지능 기반 네트워크 관리: 인공지능이 네트워크 트래픽 분석, 이상 감지, 자가 복구 등을 수행하여 운영 효율성을 높입니다.
- 웹3.0과 탈중앙화 네트워크: 블록체인 기반의 분산형 네트워크 아키텍처를 통해 데이터 주권을 사용자에게 돌려주는 방향으로 진화합니다.
