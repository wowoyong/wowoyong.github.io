## 핵심 요약

운영체제는 컴퓨터 기기 장치와 사용자 사이의 다리 역할을 합니다. 이는 컴퓨터 자원을 효율적으로 관리하고, 여러 프로그램이 동시에 원활하게 실행되도록 돕습니다. 중앙 처리 장치, 기억 장치, 입출력 장치 같은 핵심 자원을 체계적으로 할당하고 제어합니다. 사용자에게는 편리한 컴퓨터 이용 환경을 제공하며, 개발자에게는 안정적인 실행 기반을 제공합니다. 따라서 컴퓨터 체계의 성능과 안정성을 결정하는 매우 중요한 소프트웨어입니다. 현대 컴퓨터 이용 환경에서 운영체제의 역할은 점점 더 중요해지고 있습니다.

## 운영체제란?

- 컴퓨터 기기 장치와 사용자 또는 응용 프로그램 사이의 연결부 역할을 수행합니다.
- 중앙 처리 장치, 기억 장치, 저장 장치, 입출력 장치 등 컴퓨터 자원을 관리합니다.
- 여러 프로그램이 동시에 실행될 수 있도록 제어하고 조율합니다.
- 사용자에게 편리하고 효율적인 컴퓨터 이용 환경을 제공합니다.
- 체계의 안정성과 효율성을 보장하는 컴퓨터의 핵심 소프트웨어입니다.
- 컴퓨터가 켜질 때 가장 먼저 주기억장치에 로드되어 동작을 시작합니다.
- 핵심부를 중심으로 파일 체계, 입출력 관리자 등 다양한 단위로 구성됩니다.
- 컴퓨터의 모든 동작을 총괄하고 지휘하는 관리자 역할을 담당합니다.

## 핵심 기능

- **처리 과정 관리**: 프로그램의 실행 및 제어, 중앙 처리 장치 일정 계획을 담당합니다.
- **기억 장치 관리**: 주기억장치 할당, 해제, 보호 및 가상 기억 장치 구현을 처리합니다.
- **저장 장치 관리**: 파일 체계를 통해 자료를 저장, 접근 제어하고 원반형 저장 장치 일정 계획을 수행합니다.
- **입출력 관리**: 입출력 장치와 프로그램 간의 통신을 중재하고 효율적으로 제어합니다.
- **보안 및 보호**: 체계 자원에 대한 무단 접근을 방지하고 자료의 무결성을 지킵니다.
- **통신망 기능**: 통신망 통신 기능을 제공하여 다른 체계와의 연결을 가능하게 합니다.
- **사용자 연결부**: 셸(명령 줄 연결부) 또는 그래픽 사용자 연결부를 제공합니다.
- **오류 처리**: 체계 오류 발생 시 적절한 조치를 취하고 사용자에게 보고합니다.

## 처리 과정 및 스레드

- **처리 과정**: 실행 중인 프로그램의 개체로, 독립적인 자원 집합(기억 장치, 파일 등)을 가집니다.
- **스레드**: 처리 과정 내에서 실행되는 실행 단위로, 동일 처리 과정의 자원을 공유합니다.
- **처리 과정 일정 계획**: 중앙 처리 장치 시간을 여러 처리 과정에 효율적으로 분배하는 기법입니다.
- 처리 과정은 생성, 준비, 실행, 대기, 종료 등의 다양한 상태를 가집니다.
- **문맥 교환 (상황 전환)**: 중앙 처리 장치가 다른 처리 과정으로 넘어갈 때 현재 처리 과정의 상태를 저장하고 새 처리 과정의 상태를 복원하는 과정입니다.
- **스레드 사용 이유**: 자원 공유로 인한 효율성, 응답성 향상, 동시성 증대를 꾀할 수 있습니다.
- **경쟁 상태 (경주 조건)**: 여러 스레드가 공유 자원에 동시에 접근하여 예상치 못한 결과가 발생하는 현상입니다.
- **동기화 기법**: 뮤텍스, 세마포어 등을 이용하여 공유 자원 접근을 제어하고 경쟁 상태를 방지합니다.

## 기억 장치 관리

- **주기억장치 (주 기억 장치)**: 중앙 처리 장치가 직접 접근하는 빠른 저장 공간으로, 프로그램 실행에 필수적입니다.
- **가상 기억 장치**: 실제 주기억장치보다 큰 기억 장치를 사용하는 것처럼 보이게 하여 효율성을 높이는 기술입니다.
- **페이지 분할**: 주기억장치와 가상 주소 공간을 고정 크기의 페이지 단위로 나누어 관리하는 기법입니다.
- **영역 분할**: 주기억장치를 가변 크기의 논리적 단위인 구간으로 나누어 관리하는 방식입니다.
- **교환**: 주기억장치와 보조기억장치 간에 처리 과정 전체 또는 일부를 이동시키는 기법입니다.
- **페이지 교체 연산 절차**: 주기억장치 공간이 부족할 때 어떤 페이지를 보조기억장치로 내릴지 결정하는 방법입니다.
- **외부 단편화**: 가변 분할 방식에서 사용 가능한 기억 장치 구획들이 너무 작게 흩어져 전체 공간이 있어도 할당하기 어려운 현상입니다.
- **내부 단편화**: 고정 분할 방식에서 할당된 기억 장치 공간이 실제 사용량보다 커서 남는 공간이 발생하는 낭비입니다.

## 파일 체계

- **파일**: 논리적인 정보 저장 단위로, 사용자가 접근하는 가장 기본적인 자료 형태입니다.
- **파일 목록**: 파일들을 계층적으로 조직하고 관리하는 구조로, 파일의 위치 정보를 담습니다.
- **파일 접근 방식**: 순차 접근, 직접 접근, 색인 접근 등 자료 읽기/쓰기 방식을 결정합니다.
- **파일 할당 방식**: 연속 할당, 연결 목록 할당, 색인 할당 등 원반형 저장 장치에 파일을 저장하는 방법을 의미합니다.
- **파일 속성**: 이름, 유형, 위치, 크기, 접근 권한, 생성 및 수정 시간 등 파일에 대한 초자료입니다.
- **입출력 체계**: 기기 장치에 독립적인 입출력 연결부를 제공하여 프로그램 개발을 용이하게 합니다.
- **원반형 저장 장치 일정 계획**: 원반형 저장 장치 머리부의 이동을 최적화하여 원반형 저장 장치 입출력의 효율성을 증대시키는 기법입니다.
- **임시 저장**: 자주 접근하는 자료를 더 빠른 저장 공간에 임시 저장하여 체계 성능을 향상시킵니다.

## 보안 및 보호

- **보호**: 체계 내 자원의 접근을 제어하여 프로그램의 오작동이나 사용자의 오용을 방지합니다.
- **보안**: 외부 공격으로부터 체계의 무결성, 기밀성, 가용성을 유지하는 광범위한 개념입니다.
- **권한 관리**: 사용자 및 처리 과정에 대한 자원 접근 권한을 부여하고 회수하는 기능을 제공합니다.
- **인증 및 인가**: 사용자의 신원을 확인(인증)하고, 확인된 사용자에게 특정 자원에 대한 접근 권한을 부여(인가)합니다.
- **접근 제어 목록 (ACL)**: 각 자원에 접근 가능한 사용자 및 그 권한 목록을 명시하는 방법입니다.
- **링 보호**: 권한 수준에 따라 실행 영역을 나누어 하위 권한 영역이 상위 권한 영역을 침범하지 못하도록 보호합니다.
- **방화벽**: 통신망 경계에서 비인가된 접근이나 악성 정보량을 차단하는 보안 체계입니다.
- **암호화**: 자료를 알아볼 수 없는 형태로 변환하여 기밀성을 유지하는 핵심 보안 기술입니다.

## 최신 흐름

- **클라우드 운영체제**: 분산 환경에서 자원을 효율적으로 관리하는 가상화 기반 운영체제 연구가 활발합니다.
- **컨테이너 기술**: 도커, 쿠버네티스와 같은 경량 가상화 기술이 응용 프로그램 배포 및 관리의 표준으로 자리매김했습니다.
- **사물 인터넷 (IoT) 운영체제**: 저전력, 경량화된 운영체제가 다양한 사물 인터넷 장치에 필수적으로 탑재되고 있습니다.
- **보안 강화**: 제로 트러스트 구조, 기기 장치 기반 보안 단위 통합 등 운영체제 보안 기능이 고도화되고 있습니다.
- **가장자리 연산 운영체제**: 자료 처리 지연을 줄이기 위해 가장자리 장치에 최적화된 운영체제 개발이 가속화되고 있습니다.
- **인공지능 통합**: 운영체제 내부에서 자원 일정 계획, 오류 예측 등에 인공지능 기법을 활용하려는 시도가 늘고 있습니다.
- **프로그래밍 언어 변화**: 러스트와 같이 기억 장치 안전성을 강조한 언어가 운영체제 핵심부 개발에 도입될 가능성이 논의됩니다.
- **이동형 운영체제 진화**: 안드로이드, iOS의 기능 확장과 사용자 경험 개선이 지속적으로 이루어지고 있습니다.
- **분산 운영체제**: 여러 컴퓨터 자원을 단일 체계처럼 보이게 하는 분산 처리 기술이 더욱 발전하고 있습니다.
- **차세대 파일 체계**: 성능, 안정성, 확장성을 높인 새로운 파일 체계 (예: ZFS, Btrfs) 연구 및 적용이 확대됩니다.

## 가상화

- **가상화**: 단일 물리적 기기 장치 위에서 여러 개의 독립적인 가상 체계를 실행하는 기술입니다.
- **초감시자**: 가상 기기를 생성하고 관리하는 소프트웨어 계층으로, 가상 기기 감시자라고도 불립니다.
- **가상 기기 (VM)**: 실제 컴퓨터처럼 동작하는 소프트웨어 기반의 가상 환경으로, 독립적인 운영체제를 가질 수 있습니다.
- **전가상화**: 초감시자가 물리적 기기 장치 자원을 완전히 모의 재현하여 가상 기기에 제공하는 방식입니다.
- **반가상화**: 가상 기기 운영체제가 초감시자와 직접 통신하도록 수정되어 성능을 향상시키는 방식입니다.
- **컨테이너 가상화**: 운영체제 핵심부를 공유하며 응용 프로그램과 그 종속성을 격리하는 경량 가상화 기술입니다 (예: 도커).
- **장점**: 자원 효율성 증대, 격리 통한 안정성, 재해 복구 용이, 개발 및 시험 환경 구축에 유용합니다.
- **클라우드 연산의 핵심 기반 기술**: 유연한 자원 할당과 확장성을 가능하게 합니다.
