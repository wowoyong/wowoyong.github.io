## 오늘의 면접 주제
> React와 JavaScript 기반 생태계가 계속 주목받는 지금, 클로저는 프론트엔드와 Node.js 면접에서 가장 자주 나오는 핵심 개념입니다.

## 핵심 한 줄 요약
클로저는 함수가 선언될 때의 스코프를 기억해서, 함수 바깥 변수를 함수 실행 이후에도 계속 참조할 수 있게 하는 동작입니다.

## 면접 질문 & 모범 답변

### Q1. 클로저란 무엇인가요?
**기본 답변**: 클로저는 함수와 그 함수가 선언될 때의 렉시컬 환경을 함께 묶어서 보는 개념입니다. 그래서 외부 함수 실행이 끝난 뒤에도 내부 함수는 외부 변수에 접근할 수 있습니다.  
**심화 포인트**: 단순히 "함수가 변수를 기억한다"로 끝내면 약합니다. 정확히는 함수가 값 자체를 복사하는 것이 아니라, 선언 시점의 스코프 체인을 따라 변수에 접근할 수 있는 환경을 유지하는 것입니다.

### Q2. 클로저는 왜 동작하나요?
**기본 답변**: JavaScript는 렉시컬 스코프를 사용합니다. 변수 접근 범위가 함수가 호출된 위치가 아니라, 함수가 선언된 위치로 결정되기 때문에 클로저가 성립합니다.  
**심화 포인트**: 실행 시점이 아니라 선언 시점이 기준입니다. 그래서 같은 함수를 다른 곳에서 호출해도, 참조하는 외부 변수 범위는 바뀌지 않습니다.

```js
function outer() {
  let count = 0;
  return () => ++count;
}
```

### Q3. 클로저와 스코프는 어떤 관계인가요?
**기본 답변**: 스코프는 변수에 접근할 수 있는 범위입니다. 클로저는 그 스코프 규칙을 함수가 계속 활용하는 결과입니다.  
**심화 포인트**: 스코프는 규칙이고, 클로저는 그 규칙 덕분에 나타나는 현상입니다. 둘을 같은 말처럼 섞어 말하면 개념 정리가 덜 된 것으로 보일 수 있습니다.

### Q4. 실무에서 클로저를 어디에 쓰나요?
**기본 답변**: 상태를 은닉할 때 자주 씁니다. 또 커링, 함수 팩토리, 이벤트 핸들러, 메모이제이션 같은 패턴에도 많이 사용합니다.  
**심화 포인트**: class의 private 필드가 없던 시절에는 클로저가 사실상 캡슐화 도구였습니다. 지금도 라이브러리 코드나 함수형 코드에서는 여전히 많이 씁니다.

### Q5. 커링과 메모이제이션에서 클로저는 어떻게 쓰이나요?
**기본 답변**: 커링은 일부 인자를 먼저 고정한 새 함수를 만드는 방식입니다. 메모이제이션은 이전 계산 결과를 기억해 다시 계산하지 않는 방식인데, 둘 다 외부 변수를 기억해야 하므로 클로저를 사용합니다.  
**심화 포인트**: 커링은 재사용성과 조합성을 높이고, 메모이제이션은 성능을 높입니다. 면접에서는 "왜 쓰는가"까지 연결해서 말해야 좋습니다.

```js
const add = x => y => x + y;
const add10 = add(10);
```

### Q6. 클로저가 메모리 누수를 만들 수 있나요?
**기본 답변**: 가능합니다. 더 이상 필요 없는 큰 객체나 DOM 참조를 클로저가 계속 잡고 있으면 GC 대상이 되지 않을 수 있습니다.  
**심화 포인트**: 클로저 자체가 문제는 아닙니다. 필요 없는 참조를 오래 유지하는 코드가 문제입니다. 면접에서는 "클로저 = 누수"라고 말하면 오해입니다.

### Q7. `var`와 `let`은 클로저와 어떤 차이를 만들까요?
**기본 답변**: `var`는 함수 스코프이고, `let`은 블록 스코프입니다. 반복문에서 `var`를 쓰면 하나의 변수를 여러 함수가 같이 참조해서 예상과 다른 결과가 나올 수 있습니다.  
**심화 포인트**: 주니어 면접에서 자주 나오는 함정입니다. `for`문 안 비동기 콜백에서 같은 값만 출력되는 이유를 클로저와 스코프로 설명할 수 있어야 합니다.

### Q8. 면접에서 클로저를 설명할 때 가장 중요한 포인트는 무엇인가요?
**기본 답변**: 정의만 말하지 말고, 렉시컬 스코프와 실제 활용 예시를 함께 설명해야 합니다. 그리고 장점뿐 아니라 메모리 관리 이슈까지 말하면 답변이 단단해집니다.  
**심화 포인트**: 좋은 답변은 개념, 동작 원리, 활용, 주의점이 한 흐름으로 이어집니다. 암기한 정의만 말하면 깊이가 부족해 보입니다.

## 헷갈리기 쉬운 포인트
클로저는 값을 복사해서 저장하는 개념이 아닙니다. 외부 변수에 접근할 수 있는 환경을 유지하는 개념입니다.  
스코프와 클로저는 같은 말이 아닙니다. 스코프는 범위 규칙이고, 클로저는 그 규칙이 실제 함수에서 드러난 결과입니다.  
메모리 누수는 클로저 때문이 아니라, 불필요한 참조를 계속 유지하는 코드 때문에 생깁니다.

## 면접관 시각
이 주제에서는 정의 암기보다 원리 이해를 봅니다. 렉시컬 스코프를 설명하지 못하면 클로저를 제대로 이해했다고 보기 어렵습니다. 실무 예시를 간단히 들고, 메모리 관리나 `var` 반복문 문제까지 연결하면 실제로 써본 사람처럼 보입니다.
