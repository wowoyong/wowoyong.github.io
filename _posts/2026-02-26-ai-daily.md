### TeamOut, AI 에이전트로 회사 리트릿 계획 자동화

## GitHub 하이라이트

### [ollama/ollama](https://github.com/ollama/ollama)
⭐ 163,443 | Go

로컬 머신에서 Kimi-K2.5, GLM-5, MiniMax, DeepSeek, Qwen, Gemma 등 다양한 오픈소스 LLM을 쉽게 실행할 수 있는 도구입니다. GPU 메모리 제약 없이 여러 모델을 빠르게 실험할 수 있습니다.

**주목 이유**: AI 개발자들이 가장 선호하는 로컬 LLM 런타임으로, 엔터프라이즈급 생산성을 제공합니다.

### [huggingface/transformers](https://github.com/huggingface/transformers)
⭐ 157,004 | Python

Hugging Face의 대표 라이브러리로, 텍스트, 이미지, 오디오, 멀티모달 모델의 정의, 학습, 추론을 통일된 프레임워크로 제공합니다. 최신 SOTA 모델들을 즉시 사용할 수 있습니다.

**주목 이유**: 학계와 업계를 가장 광범위하게 연결하는 핵심 표준 라이브러리입니다.

### [f/prompts.chat](https://github.com/f/prompts.chat)
⭐ 147,902 | HTML

커뮤니티가 공유하는 ChatGPT 프롬프트 모음집으로, 개인 또는 조직 내에서 셀프호스팅할 수 있습니다. 프라이버시를 지키면서 검증된 프롬프트를 활용할 수 있습니다.

**주목 이유**: 오픈소스 기반 프롬프트 관리의 최강자로, 엔터프라이즈 배포에 이상적입니다.

### [langgenius/dify](https://github.com/langgenius/dify)
⭐ 130,410 | TypeScript

에이전트 워크플로우 개발을 위한 프로덕션 준비 플랫폼으로, 노코드/로우코드 방식으로 복잡한 AI 파이프라인을 구축할 수 있습니다. 비즈니스 로직과 AI를 통합하기에 최적화되어 있습니다.

**주목 이유**: 실제 프로덕션 배포를 목표로 설계된 엔터프라이즈급 AI 플랫폼입니다.

### [langchain-ai/langchain](https://github.com/langchain-ai/langchain)
⭐ 127,502 | Python

LLM 기반 에이전트와 체인 구축의 표준 프레임워크로, 메모리, 도구 통합, 프롬프트 관리 등을 일관되게 처리합니다. Python 에코시스템에서 가장 인기 있는 선택입니다.

**주목 이유**: AI 에이전트 개발의 사실상 표준 라이브러리로 자리 잡았습니다.

### [open-webui/open-webui](https://github.com/open-webui/open-webui)
⭐ 124,967 | Python

Ollama, OpenAI API 등 다양한 백엔드를 지원하는 사용자 친화적 AI 인터페이스입니다. 로컬 LLM과 클라우드 모델을 동일한 UI에서 관리할 수 있습니다.

**주목 이유**: 로컬-클라우드 하이브리드 LLM 환경의 통합 관리 솔루션으로 주목받고 있습니다.

### [firecrawl/firecrawl](https://github.com/firecrawl/firecrawl)
⭐ 85,873 | TypeScript

웹사이트 전체를 LLM 친화적인 마크다운 또는 구조화된 데이터로 변환해 주는 웹 데이터 API입니다. RAG 시스템의 데이터 수집 단계를 크게 간소화합니다.

**주목 이유**: AI 에이전트의 웹 정보 수집 능력을 획기적으로 향상시키는 도구입니다.

### [browser-use/browser-use](https://github.com/browser-use/browser-use)
⭐ 79,019 | Python

AI 에이전트가 웹사이트와 상호작용하도록 해 주는 자동화 라이브러리입니다. 클릭, 입력, 스크롤 등 일반적인 사용자 행동을 프로그래밍할 수 있습니다.

**주목 이유**: AI 에이전트의 웹 자동화 능력을 단순하게 구현할 수 있는 핵심 도구입니다.

## 주목할 논문

### Latent Context Compilation: 긴 컨텍스트를 컴팩트한 포터블 메모리로 압축

장기 컨텍스트 LLM 배포의 효율성을 높이기 위해 압축과 테스트 시간 학습 사이의 딜레마를 해결합니다. 모델 가중치를 수정하지 않으면서도 분포 외 일반화 능력을 유지하는 새로운 접근법을 제시합니다.

📄 [arXiv](https://arxiv.org/abs/2602.21221)

### ACAR: 감사 가능한 결정 추적을 가진 멀티모델 앙상블의 적응형 복잡도 라우팅

여러 LLM 모델을 상황에 맞게 조합하여 활용하는 오케스트레이션 프레임워크입니다. 자체 일관성 분산을 활용해 작업 복잡도에 따라 단일, 이중, 삼중 모델 실행을 자동으로 선택합니다.

📄 [arXiv](https://arxiv.org/abs/2602.21231)

### Urban Vibrancy Embedding과 교통 예측 응용

실시간 유동 인구 데이터에서 도시 활력 임베딩을 추출해 교통 예측을 개선하는 접근법입니다. 모바일 데이터로부터 도시의 동적 인간 활동을 캡처하는 새로운 방식을 제안합니다.

📄 [arXiv](https://arxiv.org/abs/2602.21232)

### AngelSlim: 대형 모델 압축을 위한 접근 가능하고 효율적인 툴킷

Tencent Hunyuan 팀이 개발한 포괄적 모델 압축 도구로, 양자화, 추측 디코딩, 토큰 프루닝, 증류 등 최신 알고리즘을 통합했습니다. 개발자와 연구자를 위한 통일된 인터페이스를 제공합니다.

📄 [arXiv](https://arxiv.org/abs/2602.21233)

### Soft Set 이론 및 확장의 동적 서베이

파라미터화된 의사결정을 위한 프레임워크로서의 Soft Set 이론과 그 수십 년간의 발전을 정리한 종합 리뷰입니다. 불확실성을 구조화된 방식으로 모델링하는 수학적 기초를 제공합니다.

📄 [arXiv](https://arxiv.org/abs/2602.21268)

### Group Orthogonalized Policy Optimization: Hilbert 공간의 직교 투영으로서의 정책 최적화

LLM 정렬을 위한 새로운 알고리즘으로, Hilbert 함수 공간의 기하학을 활용합니다. 확률 심플렉스 대신 Hilbert 공간에서 최적화하여 지수 곡률의 한계를 극복합니다.

📄 [arXiv](https://arxiv.org/abs/2602.21269)

### 신경망 최적화 전략과 손실 지형의 지형학

다중 차원 매개변수 최적화와 비볼록 손실 지형의 관계를 탐구합니다. 훈련 데이터에서 잘 수행하는 저손실 영역의 특성과 일반화 성능 간의 연관성을 분석합니다.

📄 [arXiv](https://arxiv.org/abs/2602.21276)

### Robust AI Evaluation through Maximal Lotteries: 최대 로또리를 통한 견고한 AI 평가

언어 모델의 주관적 작업 평가를 위해 쌍별 비교와 Bradley-Terry 순위 방식의 한계를 지적합니다. 이질적 평가자 선호도를 하나의 순서로 강제하지 않는 새로운 평가 방법론을 제안합니다.

📄 [arXiv](https://arxiv.org/abs/2602.21297)

## 오늘의 트렌드 요약

2월 26일의 AI 생태계는 **로컬 AI 민주화와 엔터프라이즈 AI 자동화**라는 두 축을 중심으로 움직이고 있습니다. ollama와 open-webui 같은 로컬 LLM 도구들이 계속 강세를 보이면서, 개발자들이 API 비용 없이 모델을 실험할 수 있는 환경이 정착되고 있습니다. 동시에 Dify, LangChain 같은 워크플로우 플랫폼과 firecrawl, browser-use 같은 에이전트 도구들의 별 수가 가파르게 증가하며, **AI 에이전트를 실제 업무에 적용하려는 움직임이 본격화**되고 있습니다. 학술 분야에서는 장기 컨텍스트 효율화, 멀티모델 오케스트레이션, LLM 정렬 등 **생산성과 안정성을 동시에 확보하는 기술**들이 집중 연구되고 있는 상황입니다.
