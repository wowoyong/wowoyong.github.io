## 핵심 요약

알고리즘은 입력을 받아 원하는 출력을 만들기 위한 유한한 절차이며, 정확성뿐 아니라 시간과 메모리 사용까지 함께 평가해야 한다. 학부 수준에서는 정렬, 탐색, 분할 정복, 동적 계획법, 그래프 탐색처럼 반복해서 등장하는 설계 틀을 익히는 것이 중요하다. 실제 적용에서는 같은 문제라도 데이터 크기, 실시간 요구, 메모리 제한에 따라 적합한 알고리즘이 달라지므로 점근 표기와 자료구조 선택을 함께 봐야 한다. 예를 들어 작은 입력에서는 단순한 방법이 더 빠를 수 있지만, 대규모 서비스에서는 로그 시간이나 선형 시간 차이가 비용과 응답 속도를 크게 바꾼다. 최근에는 인공지능 시스템, 대규모 그래프, 개인정보 보호, 스트리밍 처리, 에너지 효율 같은 환경 변화에 맞춰 알고리즘 연구 방향도 확장되고 있다. 그래서 시험 대비뿐 아니라 실제 시스템 설계 관점에서 알고리즘을 이해하는 것이 중요하다.

## 알고리즘 평가 기준과 점근 표기 읽기

- 알고리즘은 단순히 답을 내는 절차가 아니라, 제한된 시간과 메모리 안에서 답을 내는 방법까지 포함해 평가한다. 예를 들어 같은 정렬이라도 입력이 백만 개면 차이가 크게 난다.
- 시간 복잡도는 입력 크기 증가에 따라 연산 수가 어떻게 늘어나는지 본다. 실무에서는 응답 지연을 예측하는 기준으로 자주 쓴다.
- 공간 복잡도는 추가 메모리 사용량을 뜻한다. 병합 정렬은 빠르지만 추가 배열이 필요해 메모리 제약 환경에서는 부담이 될 수 있다.
- 빅 오 표기는 상한을 거칠게 보여 주며, 최악 상황 중심으로 비교할 때 유용하다. 시험에서는 상수항과 낮은 차수항을 보통 무시한다.
- 빅 오메가는 하한, 빅 세타는 상한과 하한이 같은 정도를 뜻한다. 따라서 빅 세타가 붙으면 증가율을 더 정확히 설명한다고 볼 수 있다.
- 최악 시간만 보면 부족할 때도 있다. 예를 들어 해시 탐색은 최악에는 느릴 수 있지만 평균적으로 매우 빨라서 실제 서비스에서 널리 쓴다.
- 점근 표기가 같아도 실제 속도는 다를 수 있다. 입력이 작으면 단순 반복문 기반 방법이 복잡한 고급 알고리즘보다 더 빠른 경우도 많다.
- 따라서 문제를 풀 때는 점근 표기, 상수 비용, 메모리 사용, 구현 난도를 함께 비교해야 한다. 이것이 이론과 실무를 연결하는 핵심 해석이다.

## 정렬과 탐색에서 배우는 기본 설계 감각

- 선형 탐색은 처음부터 끝까지 확인하는 단순한 방법이고, 이진 탐색은 정렬된 배열에서 범위를 반씩 줄인다. 정렬 여부가 알고리즘 선택을 바꾸는 대표 사례다.
- 버블 정렬과 선택 정렬은 이해는 쉽지만 큰 입력에는 비효율적이다. 교육용으로는 좋지만 실무용 기본 선택지는 아니다.
- 삽입 정렬은 거의 정렬된 데이터에서 강하다. 예를 들어 앞부분만 조금 흐트러진 목록을 다룰 때 예상보다 빠르게 동작한다.
- 병합 정렬은 안정적이고 시간 복잡도가 일정해 대용량 데이터 처리에 유리하다. 대신 추가 메모리가 필요하다는 점이 교환 조건이다.
- 퀵 정렬은 평균적으로 매우 빠르고 구현도 실용적이지만, 축 선택이 나쁘면 성능이 급격히 떨어질 수 있다. 그래서 실제 구현은 축 선택 전략이 중요하다.
- 힙 정렬은 추가 메모리 없이 안정적인 성능을 기대할 수 있지만, 실제 상수 비용 때문에 퀵 정렬보다 느리게 느껴질 수 있다.
- 해시 탐색은 키를 바로 위치로 보내 빠르지만, 충돌 처리 방식에 따라 성능이 달라진다. 따라서 자료구조와 함께 이해해야 한다.
- 정렬과 탐색은 단독 주제가 아니라 다른 문제의 전처리 단계로 자주 쓰인다. 예를 들어 중복 제거, 범위 질의, 우선순위 처리의 출발점이 된다.

## 분할 정복과 재귀로 큰 문제를 쪼개는 방법

- 분할 정복은 큰 문제를 작은 하위 문제로 나누고, 각각을 해결한 뒤 결과를 합치는 방식이다. 병합 정렬이 가장 대표적인 예시다.
- 재귀는 함수가 자기 자신을 호출하는 구조이며, 분할 정복을 코드로 표현할 때 자연스럽다. 다만 종료 조건이 없으면 무한 호출이 된다.
- 분할 정복의 장점은 구조가 명확하다는 점이다. 문제를 반씩 줄이면 깊이가 로그 수준으로 감소해 성능 분석이 쉬워진다.
- 모든 재귀가 효율적인 것은 아니다. 피보나치 수를 단순 재귀로 계산하면 같은 부분 문제를 반복해서 매우 느려진다.
- 병합 정렬은 분할 후 합치는 비용이 크고, 이진 탐색은 분할만 하고 합치는 비용이 거의 없다. 같은 분할 정복이라도 비용 구조가 다르다.
- 마스터 정리는 분할 정복 점화식을 빠르게 해석하는 도구다. 시험에서는 하위 문제 개수, 크기 감소 비율, 합치는 비용을 먼저 본다.
- 실무에서는 재귀 깊이 제한도 고려해야 한다. 입력이 매우 크면 반복문으로 바꾸거나 명시적 스택을 사용하는 편이 안전할 수 있다.
- 핵심은 문제를 어떻게 쪼개면 중복이 줄고 결합이 쉬운지 찾는 것이다. 이것이 분할 정복 설계 감각의 본질이다.

## 동적 계획법과 탐욕법을 구분하는 기준

- 동적 계획법은 겹치는 부분 문제와 최적 부분 구조가 있을 때, 중복 계산을 저장해 전체 최적해를 구하는 방법이다. 배낭 문제나 편집 거리에서 자주 본다.
- 탐욕법은 매 단계에서 가장 좋아 보이는 선택을 즉시 하는 방식이다. 구현이 단순하고 빠르지만 항상 정답을 보장하지는 않는다.
- 예를 들어 거스름돈 문제는 화폐 단위에 따라 탐욕법이 맞을 수도 있고 틀릴 수도 있다. 따라서 직관만으로 선택하면 위험하다.
- 동적 계획법은 보통 표를 채우거나 메모이제이션을 사용한다. 시간은 줄지만 공간이 커질 수 있어 메모리 최적화가 자주 따라온다.
- 최장 증가 부분 수열은 단순 동적 계획법으로도 풀 수 있지만, 더 나은 방법으로 개선 가능하다. 같은 문제도 접근 수준이 여러 단계다.
- 탐욕법이 맞는지 확인하려면 교환 논법이나 선택의 안전성을 증명해야 한다. 최소 신장 트리 알고리즘이 대표 예다.
- 시험에서는 점화식만 외우지 말고 상태 정의를 정확히 세워야 한다. 상태를 잘못 잡으면 점화식이 맞아도 구현이 무너진다.
- 실무 관점에서는 동적 계획법은 최적화 문제, 탐욕법은 빠른 의사결정 문제에 강하다. 다만 요구 정확도와 입력 규모를 같이 봐야 한다.

## 그래프 알고리즘으로 관계와 경로를 다루기

- 그래프는 정점과 간선으로 관계를 표현하는 구조다. 도로망, 소셜 관계, 선수과목 연결처럼 실제 모델링 범위가 매우 넓다.
- 너비 우선 탐색은 가까운 순서대로 확장하고, 깊이 우선 탐색은 한 방향으로 끝까지 내려간다. 최단 간선 수 경로는 보통 너비 우선 탐색이 유리하다.
- 다익스트라는 음수 가중치가 없을 때 단일 시작점 최단 경로를 구한다. 내비게이션의 기본 아이디어로 자주 비유된다.
- 벨만 포드는 음수 가중치를 처리할 수 있지만 더 느리다. 즉, 더 넓은 상황을 다루는 대신 성능을 양보하는 비교가 된다.
- 플로이드 워셜은 모든 정점 쌍 최단 경로를 한 번에 구할 수 있다. 정점 수가 작고 질의가 많을 때 해석이 편하다.
- 최소 신장 트리는 모든 정점을 최소 비용으로 연결하는 문제다. 통신망 설치 비용 최소화 같은 상황에서 직접 연결된다.
- 위상 정렬은 선행 관계가 있는 작업 순서를 잡을 때 쓴다. 교과목 수강 순서나 빌드 순서 계산이 대표 예시다.
- 그래프 문제는 표현 방식이 성능을 좌우한다. 인접 행렬은 접근이 빠르지만 메모리를 많이 쓰고, 인접 리스트는 희소 그래프에서 더 실용적이다.

## 무작위화와 근사 알고리즘의 현실적 가치

- 무작위화 알고리즘은 난수를 이용해 평균 성능을 높이거나 구조적 편향을 줄인다. 퀵 정렬의 무작위 축 선택이 대표적이다.
- 이 방법의 장점은 입력이 악의적으로 구성되어도 성능 저하를 줄일 수 있다는 점이다. 실무 서비스에서 예측 가능성을 높이는 데 도움이 된다.
- 확률적으로만 맞는 알고리즘과 항상 맞되 시간이 달라지는 알고리즘은 구분해야 한다. 둘 다 무작위화를 쓰지만 의미가 다르다.
- 근사 알고리즘은 최적해를 빨리 구하기 어려운 문제에서, 충분히 좋은 해를 빠르게 구하는 전략이다. 이동 경로 최적화 문제가 대표적이다.
- 정확한 최적해가 중요하지 않은 물류, 광고 배치, 서버 할당에서는 근사해가 더 현실적일 수 있다. 계산 시간 자체가 비용이기 때문이다.
- 성능 보장 비율은 근사해가 최적해에서 얼마나 떨어지는지 설명한다. 따라서 단순한 휴리스틱보다 이론적 신뢰도가 높다.
- 휴리스틱은 실전에서 자주 쓰이지만, 항상 품질 보장이 있지는 않다. 근사 알고리즘과 구분해서 보는 습관이 중요하다.
- 결국 무작위화와 근사는 완벽함보다 확장성과 속도를 우선하는 접근이다. 대규모 계산 환경에서 특히 가치가 커진다.

## 시험과 코딩 테스트에서 자주 틀리는 판단 지점

- 문제 유형을 서둘러 단정하면 틀리기 쉽다. 최단 경로처럼 보여도 실제로는 동적 계획법이나 이분 탐색이 핵심일 수 있다.
- 입력 크기를 먼저 보지 않으면 비효율적 풀이를 고르기 쉽다. 예를 들어 이중 반복문이 가능한지 여부는 제한 조건이 먼저 결정한다.
- 정렬이 필요한지, 해시로 충분한지 판단하는 습관이 중요하다. 중복 체크는 정렬보다 해시가 더 단순한 경우가 많다.
- 재귀 문제에서는 종료 조건과 인덱스 범위를 가장 먼저 점검해야 한다. 논리는 맞아도 경계 조건에서 자주 무너진다.
- 동적 계획법은 상태 정의와 초기값이 절반이다. 점화식만 맞고 초기값이 틀리면 전부 틀린 답이 나온다.
- 그래프 문제에서는 방향 그래프인지 무방향 그래프인지, 가중치가 있는지 없는지부터 구분해야 한다. 알고리즘 선택이 바로 달라진다.
- 시간 초과는 알고리즘 탓만이 아니다. 불필요한 복사, 느린 입출력, 비효율적 자료구조 선택도 원인이 된다.
- 풀이를 마친 뒤에는 작은 예시, 극단 입력, 빈 경우를 직접 넣어 보는 것이 좋다. 실무의 단위 시험과 비슷한 검증 습관이다.

## 최신 흐름

- 대규모 인공지능 학습과 추론에서 최적화 알고리즘의 중요성이 더 커지고 있다. 같은 모델이라도 학습 절차와 배치 전략에 따라 비용 차이가 크게 난다.
- 그래프 기반 학습이 확산되면서, 매우 큰 그래프를 빠르게 처리하는 탐색과 분할 알고리즘 연구가 활발하다. 추천 시스템과 생명 정보 처리에서 수요가 크다.
- 스트리밍 알고리즘이 다시 주목받고 있다. 모든 데이터를 저장하지 않고도 요약 통계를 유지해야 하는 로그 분석과 센서 처리에 잘 맞기 때문이다.
- 개인정보 보호를 고려한 학습 절차가 중요해지면서, 차등 개인정보 보호와 연합 학습의 최적화 알고리즘이 2025년 이후 더 활발히 연구되고 있다.
- 분산 환경에서 통신 비용을 줄이는 알고리즘 설계가 핵심 과제가 되고 있다. 계산 자체보다 장치 사이 데이터 이동이 병목이 되는 경우가 많아서다.
- 에너지 효율을 고려한 알고리즘 평가가 늘고 있다. 같은 시간 복잡도라도 메모리 접근 패턴이 전력 소비를 크게 바꾼다는 해석이 중요해졌다.
- 확률적 방법과 근사 기법이 대규모 조합 최적화에서 더 널리 쓰이고 있다. 완전한 최적해보다 빠르고 안정적인 준최적해가 산업 현장에서 선호되기 때문이다.
- 양자 계산 발전과 함께 오류에 강한 양자 알고리즘 구조 연구가 2025년과 2026년에도 중요한 흐름으로 이어지고 있다. 아직 초기 단계지만 장기적으로 파급력이 크다.
- 고전 알고리즘과 기계학습을 결합한 하이브리드 접근이 늘고 있다. 예를 들어 탐색 순서나 가지치기 기준을 학습으로 보완해 실제 성능을 높이는 방식이다.
- 최근 알고리즘 교육도 바뀌고 있다. 단순 증명 암기보다 성능 실험, 입력 특성 분석, 실제 시스템 제약을 함께 보는 방향으로 점점 이동하고 있다.
