## 핵심 요약

네트워크는 여러 장치가 데이터를 주고받기 위해 규칙, 주소 체계, 전송 방식, 장비 역할을 함께 사용하는 구조다. 학부 과정에서는 계층 구조, 주소와 이름 해석, 전송 제어, 경로 선택, 성능 지표를 연결해서 이해하는 것이 중요하다. 실제 적용에서는 웹 서비스 응답 속도, 영상 통화 품질, 클라우드 연결 안정성처럼 사용자가 체감하는 품질이 모두 네트워크 설계와 운영의 결과로 나타난다. 또한 같은 데이터라도 유선망, 무선망, 내부망, 외부망에 따라 병목 지점과 최적화 방법이 달라진다. 최근에는 와이파이 7, 5세대 고도화 특화망, 경계 컴퓨팅, 인공지능 기반 운영 자동화처럼 더 빠르고 더 예측 가능한 연결을 만드는 흐름이 강해지고 있다. 그래서 네트워크 학습은 단순한 이론 암기보다 성능과 안정성을 실제 상황에 대입해 해석하는 관점이 중요하다.

## 계층 구조로 보는 데이터 전달 경로

- 네트워크 계층 구조는 기능을 나눠 복잡도를 줄이는 방법이며, 예를 들어 웹 페이지 요청에서도 응용, 전송, 인터넷, 링크 단계가 각자 역할을 맡는다.
- 응용 계층은 사용자가 직접 만나는 서비스 규칙을 다루며, 같은 채팅이라도 메시지 형식과 인증 방식에 따라 동작 차이가 난다.
- 전송 계층은 끝단 사이 전달 품질을 조절하는데, 파일 전송처럼 정확성이 중요한 경우와 실시간 통화처럼 지연이 중요한 경우를 다르게 다룬다.
- 인터넷 계층은 목적지까지 가는 경로를 정하는 역할을 하며, 실무에서는 여러 라우터를 거쳐도 같은 목적지에 도달하도록 주소 기반 전달을 수행한다.
- 링크 계층은 바로 옆 장치와의 통신을 담당하고, 같은 건물 안 스위치 연결처럼 가까운 구간의 오류와 프레임 전달을 처리한다.
- 계층 분리는 교체와 확장을 쉽게 만든다. 예를 들어 무선 접속 방식이 바뀌어도 상위 웹 서비스 코드는 대부분 그대로 유지된다.
- 문제 해결에서도 계층 관점이 유용하다. 웹이 느릴 때 응용 서버 문제인지, 전송 재전송 문제인지, 무선 구간 간섭인지 순서대로 좁혀갈 수 있다.

## 주소 체계와 이름 해석이 연결을 성립시키는 방식

- 네트워크 주소는 장치의 위치를 식별하는 값이며, 집 주소가 배송 위치를 정하듯 패킷의 목적지를 정한다.
- 물리 주소는 같은 링크 내부 전달에 쓰이고, 논리 주소는 여러 네트워크를 건너는 데 쓰여서 사용 범위가 다르다.
- 도메인 이름 해석은 사람이 읽기 쉬운 이름을 주소로 바꾸는 과정이며, 예를 들어 사용자는 서비스 이름만 입력해도 실제 통신은 숫자 주소로 진행된다.
- 공인 주소와 사설 주소를 비교하면, 공인 주소는 인터넷에서 직접 보이고 사설 주소는 내부망에서만 쓰여 보안과 주소 절약에 유리하다.
- 주소 변환 장치는 여러 내부 장치가 하나의 외부 주소를 공유하게 해 주며, 가정용 공유기에서 가장 흔하게 볼 수 있다.
- 서브넷 분할은 큰 네트워크를 작은 단위로 나누는 기법으로, 실무에서는 부서별 방송 범위를 줄여 혼잡과 장애 확산을 막는다.
- 이름 해석 지연이 커지면 전체 서비스가 느려질 수 있으므로, 실제 운영에서는 캐시와 지역 분산으로 첫 연결 시간을 줄인다.

## 신뢰성과 지연을 가르는 전송 방식 선택

- 연결 지향 전송은 순서와 재전송을 보장해 안정적이지만, 비교하면 지연과 제어 비용이 더 크다.
- 비연결 전송은 빠르고 단순하지만 손실 복구를 기본 제공하지 않아서, 실시간 스트리밍처럼 일부 손실을 감수하는 서비스에 자주 쓰인다.
- 흐름 제어는 받는 쪽 처리 속도에 맞춰 보내는 양을 조절하는 기능이며, 느린 수신 장치가 버퍼 넘침을 일으키지 않게 한다.
- 혼잡 제어는 네트워크 전체 상태를 보고 전송량을 조절하는 기능으로, 실무에서는 여러 사용자가 동시에 몰릴 때 중요해진다.
- 재전송은 손실 복구에 효과적이지만 지나치면 지연이 커지므로, 온라인 게임에서는 무조건 재전송보다 선택적 복구가 더 적합할 수 있다.
- 순서 보장은 데이터베이스 복제처럼 정확성이 중요한 상황에 유리하고, 음성 통화에서는 늦게 도착한 패킷보다 바로 다음 음성이 더 중요하다.
- 전송 방식 선택은 서비스 목표와 연결된다. 은행 거래는 정확성이 우선이고, 실시간 화면 공유는 지연 감소가 우선이라는 식으로 해석하면 된다.

## 스위치와 라우터가 내부망과 외부망을 나누는 원리

- 스위치는 같은 내부망 안에서 목적지 장치를 찾아 프레임을 전달하며, 한 사무실 안 여러 컴퓨터 연결에 적합하다.
- 라우터는 서로 다른 네트워크 사이에서 경로를 정하므로, 내부망이 인터넷과 만나는 경계에서 핵심 장비가 된다.
- 스위치는 물리 주소 중심으로 동작하고 라우터는 논리 주소 중심으로 동작해, 처리 범위와 판단 기준이 다르다.
- 가상 근거리망은 하나의 물리 스위치를 여러 논리 네트워크처럼 나누는 방법이며, 실무에서는 개발망과 사무망을 분리할 때 유용하다.
- 기본 게이트웨이는 외부망으로 나가는 첫 출구라는 의미로 이해하면 쉽고, 설정이 틀리면 내부 통신은 되는데 인터넷만 안 되는 현상이 생긴다.
- 경로 선택 표는 어느 목적지로 갈 때 어느 다음 장치로 보낼지 정리한 지도와 같아서, 대규모 망일수록 자동 학습 방식이 중요해진다.
- 이중화 구성은 장비 하나가 고장 나도 서비스가 유지되게 만드는 방법이며, 학교 전산실이나 기업 센터망에서 필수에 가깝다.

## 지연·대역폭·혼잡으로 읽는 네트워크 성능 해석

- 대역폭은 한 번에 보낼 수 있는 양의 크기이고, 지연은 도착까지 걸리는 시간이어서 둘이 크다고 항상 체감이 좋은 것은 아니다.
- 예를 들어 대역폭이 큰 회선도 지연이 크면 원격 제어나 게임에서는 답답하게 느껴질 수 있다.
- 처리량은 실제로 전달된 유효 데이터 양이므로, 광고된 회선 속도와 실측 값이 다른 이유를 설명해 준다.
- 지터는 지연의 흔들림을 뜻하며, 영상 회의에서 화면은 괜찮은데 목소리가 끊기는 현상을 이해할 때 중요하다.
- 패킷 손실은 품질 저하의 직접 원인이며, 파일 전송에서는 속도 저하로 보이고 실시간 통화에서는 음성 끊김으로 나타난다.
- 혼잡은 동시에 너무 많은 트래픽이 몰려 대기열이 길어지는 현상으로, 출근길 도로 정체와 비슷하게 이해할 수 있다.
- 실무에서는 단순 평균보다 최대 지연, 구간별 손실, 시간대별 변화를 함께 봐야 원인을 찾을 수 있고, 그래서 지속 측정이 중요하다.

## 웹·영상·클라우드 서비스로 이어지는 응용 관점 이해

- ��� 서비스는 짧은 요청과 응답이 반복되므로 초기 연결 시간과 이름 해석 속도가 체감 품질에 큰 영향을 준다.
- 영상 스트리밍은 연속 재생이 중요해서, 순간 최고 속도보다 안정적인 처리량과 버퍼 운영이 더 중요하다.
- 영상 통화는 양방향 실시간성이 핵심이라 파일 다운로드와 달리 지연과 지터를 더 엄격하게 관리해야 한다.
- 클라우드 환경에서는 사용자가 같은 건물 안 서버가 아니라 먼 데이터 센터와 통신하므로, 경로 길이와 구간 병목이 성능을 좌우한다.
- 콘텐츠 전송망은 사용자 가까운 곳에서 데이터를 제공해 응답을 줄이는데, 인기 동영상을 지역별로 미리 배치하는 방식이 대표적이다.
- 부하 분산은 여러 서버에 요청을 나눠 장애와 과부하를 줄이며, 전자상거래 행사처럼 접속 폭증 상황에서 특히 중요하다.
- 응용 계층 최적화는 네트워크 이해와 분리되지 않는다. 같은 서비스라도 압축, 캐시, 요청 수 감소에 따라 망 부담이 크게 달라진다.

## 최신 흐름

- 와이파이 7 상용화가 빠르게 확대되면서 다중 링크 활용과 낮은 지연 특성이 고밀도 무선 환경의 핵심 경쟁력이 되고 있다.
- 웹 전송에서는 퀵 기반의 하이퍼텍스트 전송 규약 3세대 채택이 늘어 초기 연결 지연과 손실 복구 효율이 개선되는 흐름이 이어지고 있다.
- 5세대 이동통신 고도화는 단순 속도 향상보다 상향 전송 강화, 낮은 지연, 산업 현장 맞춤 연결 제공 쪽으로 무게가 이동하고 있다.
- 특화망은 공장, 물류, 병원처럼 폐쇄적이고 예측 가능한 환경에서 안정성과 보안을 높이는 현실적인 선택지로 자리 잡고 있다.
- 경계 컴퓨팅 결합이 강화되면서 데이터를 먼 중앙이 아니라 가까운 지점에서 처리해 로봇 제어와 실감형 서비스 응답을 줄이는 방향이 뚜렷하다.
- 인공지능 기반 네트워크 운영 자동화는 이상 징후 탐지, 장애 예측, 자원 배치를 자동화해 관리자의 반복 작업을 줄이는 쪽으로 확산되고 있다.
- 네트워크 기능의 가상화와 클라우드화가 진행되면서 전용 장비 중심 구조보다 소프트웨어 중심 배포와 유연한 확장이 더 중요해지고 있다.
- 양자내성암호 전환 준비가 통신 인프라에서도 본격화되어, 장기 보호가 필요한 데이터와 인증 체계부터 단계적으로 바꾸려는 움직임이 커지고 있다.
- 지상망과 위성망을 함께 쓰는 통합 연결이 주목받으면서, 재난 지역이나 오지에서도 끊김을 줄이기 위한 다중 경로 설계가 중요해지고 있다.
- 산업 자동화와 실시간 제어 수요가 늘면서 예측 가능한 지연을 보장하려는 결정론적 네트워크 연구와 적용이 점점 더 현실적인 과제가 되고 있다.
