## 핵심 요약

컴퓨터 네트워크는 여러 장치가 서로 연결되어 정보를 교환하는 기술 분야입니다. 이 분야는 데이터를 효율적으로 전송하고 공유하는 방법을 연구하며, 현대 사회의 모든 디지털 서비스의 기반이 됩니다. 학부 과정에서는 인터넷의 동작 원리부터 무선 통신, 보안, 그리고 최신 기술 동향까지 폭넓게 다룹니다. 네트워크 지식은 소프트웨어 개발자, 시스템 관리자, 보안 전문가 등 다양한 정보 기술 직무에 필수적입니다. 이 요약은 컴퓨터 네트워크의 주요 개념과 학습 방향을 제시합니다.

## 네트워크 기초 개념

- 컴퓨터 네트워크는 둘 이상의 장치가 서로 연결되어 데이터를 주고받는 시스템입니다.
- 주요 구성 요소로는 통신 장치(호스트), 연결 매체(유선, 무선), 그리고 데이터 전달 규칙(프로토콜)이 있습니다.
- 통신 장치는 컴퓨터, 서버, 스마트폰, 라우터, 스위치 등을 포함합니다.
- 연결 매체는 이더넷 케이블, 광섬유, 무선 근거리 통신망, 5세대 이동통신 등이 있습니다.
- 네트워크 규모에 따라 개인 근거리 통신망, 근거리 통신망, 도시권 통신망, 광역 통신망 등으로 분류됩니다.
- 데이터 전송 시 대역폭, 지연, 처리량, 패킷 손실률 등의 성능 지표가 중요합니다.
- 네트워크는 자원 공유, 정보 교환, 분산 처리 등 다양한 이점을 제공합니다.
- 네트워크 토폴로지(성형, 링형, 버스형, 메시형 등)는 장치 연결 방식을 나타냅니다.

## 계층 모델과 프로토콜

- 네트워크 통신 과정을 효율적으로 관리하기 위해 계층 모델을 사용합니다.
- 개방형 시스템 상호 연결 7계층 모델은 이론적 기반을 제공하며, 각 계층은 독립적인 기능을 수행합니다.
- 응용, 표현, 세션, 전송, 네트워크, 데이터 링크, 물리 계층으로 구성됩니다.
- 실제 인터넷에서 주로 사용되는 것은 전송 제어 프로토콜/인터넷 프로토콜 4/5계층 모델입니다.
- 응용, 전송, 인터넷, 네트워크 인터페이스(또는 물리/데이터 링크) 계층으로 구성됩니다.
- 각 계층은 특정 프로토콜(예: 전송 제어 프로토콜, 사용자 데이터그램 프로토콜, 인터넷 프로토콜, 이더넷)을 사용하여 통신합니다.
- 프로토콜은 통신 장치 간에 데이터를 주고받는 데 필요한 약속과 규칙의 집합입니다.
- 캡슐화와 역캡슐화 과정을 통해 데이터가 각 계층을 오가며 헤더 정보가 추가되거나 제거됩니다.

## 전송 계층

- 전송 계층은 송신 프로세스와 수신 프로세스 간의 논리적 통신을 제공합니다.
- 전송 제어 프로토콜은 신뢰성 있는 연결 지향 통신을 제공하며, 데이터 손실 없이 순서대로 전달을 보장합니다.
- 연결 설정(3방향 핸드셰이크), 흐름 제어, 혼잡 제어, 오류 복구 기능을 수행합니다.
- 사용자 데이터그램 프로토콜은 비연결형 프로토콜로, 신뢰성을 보장하지 않지만 빠르고 오버헤드가 적습니다.
- 실시간 스트리밍, 온라인 게임 등 지연에 민감한 응용 프로그램에 주로 사용됩니다.
- 포트 번호를 사용하여 여러 응용 프로그램이 동시에 네트워크를 사용할 수 있도록 프로세스를 식별합니다.
- 잘 알려진 포트 번호(예: 웹 서버 80번, 보안 셸 22번)는 특정 서비스에 예약되어 있습니다.
- 세그먼트라는 단위로 데이터를 분할하여 전송하고, 수신 시 재조립합니다.

## 네트워크 계층

- 네트워크 계층은 출발지에서 목적지까지 데이터를 전달하는 경로를 결정합니다.
- 인터넷 프로토콜은 데이터 패킷의 주소를 지정하고 라우팅하는 핵심 프로토콜입니다.
- 인터넷 프로토콜 주소는 네트워크에 연결된 각 장치를 고유하게 식별하는 논리적 주소입니다.
- 인터넷 프로토콜 버전 4는 32비트 주소 체계를 사용하며, 인터넷 프로토콜 버전 6는 주소 고갈 문제를 해결하기 위해 128비트 주소 체계를 도입했습니다.
- 라우터는 네트워크 간에 데이터 패킷을 전달하는 장치로, 라우팅 테이블을 기반으로 최적의 경로를 선택합니다.
- 라우팅 프로토콜(예: 개방형 최단 경로 우선, 경계 경로 프로토콜)은 라우터들이 경로 정보를 교환하고 업데이트하는 규칙입니다.
- 서브넷 마스크를 사용하여 인터넷 프로토콜 주소를 네트워크 부분과 호스트 부분으로 구분합니다.
- 패킷 교환 방식은 데이터를 작은 패킷으로 나누어 독립적으로 전송하는 방식으로, 자원 효율성을 높입니다.

## 데이터 링크 및 물리 계층

- 데이터 링크 계층은 직접 연결된 두 노드 간의 신뢰성 있는 데이터 전송을 담당합니다.
- 이더넷은 유선 근거리 통신망에서 가장 널리 사용되는 데이터 링크 기술 표준입니다.
- 매체 접근 제어 주소는 네트워크 인터페이스 카드에 고유하게 부여된 물리적 주소입니다.
- 주소 결정 프로토콜은 인터넷 프로토콜 주소를 매체 접근 제어 주소로 변환하는 데 사용됩니다.
- 무선 근거리 통신망의 표준인 무선 근거리 통신망은 전기전자학회 802.11 규약을 따릅니다.
- 물리 계층은 비트 스트림을 전기 신호, 광 신호, 무선 주파수 등으로 변환하여 전송 매체를 통해 전달합니다.
- 허브, 리피터 등은 물리 계층 장비이며, 스위치는 데이터 링크 계층 장비입니다.
- 다중 접근 프로토콜(예: 반송파 감지 다중 접근 및 충돌 감지, 반송파 감지 다중 접근 및 충돌 회피)은 공유 매체 사용 시 충돌을 방지합니다.

## 네트워크 보안 기초

- 네트워크 보안은 네트워크와 데이터에 대한 무단 접근, 오용, 변조, 파괴를 방지하는 것입니다.
- 방화벽은 미리 정의된 보안 규칙에 따라 네트워크 트래픽을 모니터링하고 제어하는 시스템입니다.
- 침입 탐지 시스템과 침입 방지 시스템은 비정상적인 네트워크 활동을 감지하고 차단합니다.
- 가상 사설망은 공중망을 통해 암호화된 터널을 생성하여 안전하게 사설 네트워크에 접속할 수 있게 합니다.
- 암호화는 데이터를 이해할 수 없는 형태로 변환하여 무단 접근으로부터 보호하는 핵심 기술입니다.
- 웹 보안의 중요한 요소인 전송 계층 보안 및 보안 소켓 계층은 웹 통신을 암호화합니다.
- 서비스 거부 공격은 여러 시스템을 이용하여 특정 서버나 네트워크를 마비시키는 공격입니다.
- 네트워크 주소 변환은 사설 인터넷 프로토콜 주소를 공인 인터넷 프로토콜 주소로 변환하여 네트워크 보안과 인터넷 프로토콜 주소 절약에 기여합니다.

## 최신 흐름

- 5세대 이동통신 및 6세대 이동통신 기술 개발이 가속화되어 초고속, 초저지연, 초연결 네트워크 환경을 구축하고 있습니다.
- 소프트웨어 정의 네트워킹과 네트워크 기능 가상화는 네트워크 관리의 유연성과 효율성을 극대화합니다.
- 엣지 컴퓨팅은 데이터 소스에 더 가까운 곳에서 데이터를 처리하여 지연 시간을 줄이고 대역폭 사용을 최적화합니다.
- 사물 인터넷 기기의 확산으로 방대한 수의 센서와 장치가 네트워크에 연결되어 새로운 서비스 모델을 창출합니다.
- 인공지능 기반 네트워크 관리는 트래픽 예측, 이상 탐지, 자율 복구 등 네트워크 운영의 지능화를 가능하게 합니다.
- 블록체인 기술이 분산형 네트워크 보안, 신원 인증, 사물 인터넷 데이터 관리 등에 적용되는 연구가 활발합니다.
- 양자 통신 및 양자 암호 기술은 미래 네트워크의 궁극적인 보안 강화를 목표로 개발 중입니다.
- 제로 트러스트 보안 모델은 모든 사용자 및 장치를 잠재적 위협으로 간주하여 지속적인 인증과 권한 검증을 요구합니다.
- 웹 3.0은 분산원장기술 기반의 탈중앙화된 인터넷 환경을 지향하며, 네트워크 아키텍처에도 변화를 가져옵니다.
- 공간 인터넷은 저궤도 위성을 활용하여 전 세계 어디에서나 인터넷 접속을 제공하는 혁신적인 시도입니다 (예: 스타링크).
