## 핵심 요약

운영체제는 컴퓨터의 하드웨어와 소프트웨어 자원을 관리하고 사용자에게 편리한 환경을 제공하는 핵심 시스템 소프트웨어입니다. 이는 컴퓨터 부팅부터 프로그램 실행, 파일 저장에 이르기까지 모든 과정을 조율합니다. 운영체제는 프로세스 관리, 메모리 관리, 파일 시스템 관리, 입출력 장치 제어 등 다양한 기능을 수행합니다. 효율적인 자원 할당과 보호를 통해 여러 프로그램이 동시에 안정적으로 실행되도록 돕습니다. 우리가 사용하는 모든 컴퓨터와 모바일 기기는 운영체제 없이는 작동할 수 없습니다.

## 운영체제란?

- 정의: 하드웨어와 사용자 및 응용 프로그램 간의 중재자 역할을 하는 시스템 소프트웨어입니다.
- 목표: 컴퓨터 자원을 효율적으로 관리하고 사용자에게 편리한 실행 환경을 제공하는 것입니다.
- 주요 구성 요소: 컴퓨터의 핵심 기능을 담당하는 커널과 사용자 명령을 처리하는 셸 등이 있습니다.
- 예시: 윈도우(Windows), 맥OS(macOS), 리눅스(Linux), 안드로이드(Android), 아이OS(iOS) 등이 있습니다.
- 부팅 과정: 컴퓨터 전원이 켜지면 운영체제가 주기억장치(메모리)에 적재되어 시스템을 초기화합니다.
- 자원 관리: 중앙 처리 장치(CPU), 메모리, 저장 장치, 입출력 장치 등을 효율적으로 분배하고 제어합니다.
- 프로그램 실행 환경: 응용 프로그램이 실행될 수 있도록 안정적인 기반과 필요한 서비스를 제공합니다.
- 보안 기능: 시스템 자원과 사용자 데이터를 무단 접근으로부터 보호하는 역할을 수행합니다.

## 운영체제의 핵심 기능

- 프로세스 관리: 실행 중인 프로그램을 나타내는 프로세스의 생성, 실행, 중지, 종료를 제어합니다.
- 메모리 관리: 프로그램들이 주기억장치(메모리)를 효율적으로 공유하고 사용할 수 있도록 할당 및 회수합니다.
- 파일 시스템 관리: 데이터를 파일 형태로 저장하고, 파일을 구성하고 접근하는 방법을 정의하며 권한을 관리합니다.
- 입출력 관리: 키보드, 마우스, 프린터, 디스크 등 다양한 입출력 장치의 작동을 제어하고 조정합니다.
- 네트워크 관리: 컴퓨터 간의 통신을 위한 네트워크 연결 설정 및 데이터 송수신 기능을 지원합니다.
- 자원 스케줄링: 중앙 처리 장치 시간, 메모리, 입출력 장치 등 한정된 자원을 여러 프로세스에 공정하고 효율적으로 분배합니다.
- 오류 처리: 하드웨어 고장, 소프트웨어 버그 등 시스템 오류를 감지하고 적절히 대응하여 안정성을 유지합니다.
- 보안 및 보호: 사용자 인증, 접근 제어 목록 등을 통해 시스템 자원과 사용자 데이터에 대한 무단 접근을 방지합니다.

## 프로세스 관리

- 프로세스 개념: 실행 중인 프로그램의 인스턴스이자 운영체제의 작업 단위입니다.
- 프로세스 상태: 생성, 준비, 실행, 대기, 종료 등의 여러 상태를 거치며 실행됩니다.
- 프로세스 제어 블록(PCB): 프로세스마다 할당되는 자료구조로, 프로세스 상태, 우선순위, 레지스터 값 등 정보를 담습니다.
- 스케줄링: 중앙 처리 장치 자원을 어떤 프로세스에 얼마나 할당할지 결정하는 작업입니다.
- 스케줄링 알고리즘: 선입선출(FCFS), 최단 작업 우선(SJF), 라운드 로빈(RR) 등 다양한 방식이 있습니다.
- 컨텍스트 스위칭: 중앙 처리 장치가 한 프로세스에서 다른 프로세스로 전환할 때 발생하는 오버헤드입니다.
- 스레드: 프로세스 내에서 실행되는 더 작은 실행 단위로, 하나의 프로세스가 여러 작업을 동시에 수행하게 합니다.
- 프로세스 간 통신(IPC): 파이프, 공유 메모리, 메시지 큐 등을 통해 독립적인 프로세스들이 데이터를 교환합니다.

## 메모리 관리

- 메모리 계층 구조: 레지스터, 캐시, 주 메모리, 보조 저장 장치 순으로 속도와 용량의 계층을 이룹니다.
- 논리 주소와 물리 주소: 중앙 처리 장치가 생성하는 주소(논리 주소)와 실제 주기억장치 주소(물리 주소)를 구분합니다.
- 가상 메모리: 실제 주기억장치보다 큰 주소 공간을 제공하여 효율적인 메모리 사용을 가능하게 합니다.
- 페이징: 가상 메모리를 고정된 크기의 페이지 단위로 나누어 물리 메모리에 불연속적으로 저장하는 기법입니다.
- 세그멘테이션: 가상 메모리를 논리적 의미를 가지는 가변적인 크기의 세그먼트 단위로 나누는 기법입니다.
- 페이지 교체 알고리즘: 주기억장치 공간이 부족할 때 어떤 페이지를 디스크로 보낼지 결정하는 전략입니다(예: 최소 최근 사용(LRU)).
- 스와핑: 주기억장치와 보조 저장 장치 간에 프로세스 전체 또는 일부를 이동시켜 메모리를 확장하는 기법입니다.
- 메모리 보호: 각 프로세스가 자신의 할당된 메모리 영역만을 사용하도록 하여 다른 프로세스의 침범을 방지합니다.

## 파일 시스템과 입출력 관리

- 파일 개념: 관련 정보를 하나의 논리적 단위로 묶어 저장 장치에 보관하는 것입니다.
- 파일 시스템 구조: 파일을 구성하고 관리하는 체계로, 디렉터리(폴더) 구조, 파일 속성, 데이터 블록 등으로 이루어집니다.
- 디렉터리 구조: 파일을 조직적으로 관리하기 위해 단일, 2단계, 트리 구조 등의 계층적 방식을 사용합니다.
- 파일 접근 방식: 데이터를 파일의 처음부터 순차적으로 읽는 순차 접근과 임의의 위치에 바로 접근하는 직접 접근 방식이 있습니다.
- 입출력 장치: 정보를 입력하거나 출력하는 하드웨어 장치로, 블록 장치(하드 디스크)와 문자 장치(키보드)로 구분됩니다.
- 장치 드라이버: 운영체제가 특정 하드웨어 장치를 제어하고 통신할 수 있도록 해주는 소프트웨어 모듈입니다.
- 버퍼링: 입출력 장치와 중앙 처리 장치 간의 속도 차이를 완화하기 위해 데이터를 임시 저장하는 기법입니다.
- 스풀링: 여러 사용자의 입출력 작업을 한 곳에 모아 순서대로 처리하여 입출력 효율을 높이는 기법입니다.

## 동시성 제어 및 교착 상태

- 동시성 문제: 여러 프로세스나 스레드가 공유 자원에 동시에 접근하려 할 때 발생하는 데이터 불일치 문제입니다.
- 임계 영역(Critical Section): 공유 자원에 접근하는 코드 영역으로, 한 번에 하나의 프로세스만 실행되어야 합니다.
- 상호 배제(Mutual Exclusion): 임계 영역에 한 번에 하나의 프로세스만 진입하도록 보장하는 원칙입니다.
- 세마포어: 동시성 문제를 해결하기 위한 도구로, 정수 값과 대기/신호 연산을 통해 공유 자원 접근을 제어합니다.
- 모니터: 상호 배제와 조건 동기화를 지원하는 고급 언어 수준의 동시성 제어 기법입니다.
- 교착 상태(Deadlock): 두 개 이상의 프로세스가 서로 상대방이 가진 자원을 기다리며 무한히 대기하는 상황입니다.
- 교착 상태 발생 조건: 상호 배제, 점유 및 대기, 비선점, 순환 대기 이 네 가지 조건이 동시에 만족될 때 발생합니다.
- 교착 상태 해결 전략: 교착 상태를 예방하거나, 발생 시 회피 또는 발견하여 복구하는 전략이 있습니다.

## 최신 흐름

- 클라우드 운영체제: 여러 서버와 가상 자원을 통합 관리하며, 필요에 따라 확장 가능한 분산 운영체제가 보편화되고 있습니다.
- 컨테이너 가상화: 도커(Docker)나 쿠버네티스(Kubernetes)를 기반으로 응용 프로그램을 가볍게 격리하고 배포하는 방식이 대세입니다.
- 임베디드 운영체제: 사물 인터넷(IoT) 기기, 스마트 가전 등 특정 목적의 소형 하드웨어에 최적화된 경량 운영체제의 수요가 증가하고 있습니다.
- 실시간 운영체제(RTOS): 로봇 제어, 자율 주행, 산업 자동화 등 정해진 시간 내에 작업을 반드시 완료해야 하는 시스템에 필수적입니다.
- 보안 강화: 제로 트러스트(Zero Trust) 아키텍처 도입, 하드웨어 기반 보안(TPM) 통합 등 운영체제 수준의 강력한 보안 기능이 중요해지고 있습니다.
- 인공지능(AI) 및 머신러닝(ML) 통합: 시스템 자원 스케줄링 최적화, 이상 탐지, 성능 예측 등에 인공지능 기술이 활용됩니다.
- 분산 운영체제: 여러 독립적인 컴퓨터가 마치 하나의 시스템처럼 작동하도록 하는 운영체제가 대규모 서비스에 적용됩니다.
- 마이크로커널 아키텍처: 커널의 핵심 기능만 남기고 대부분의 서비스를 사용자 공간으로 옮겨 안정성과 유연성을 높이는 시도가 이어집니다.
- 엣지 컴퓨팅 운영체제: 데이터 처리 및 분석을 중앙 클라우드가 아닌 데이터 발생 지점 근처에서 수행하는 데 특화된 운영체제가 부상하고 있습니다.
- 양자 컴퓨팅 운영체제 연구: 미래 양자 컴퓨터의 독특한 아키텍처와 연산 방식에 맞는 새로운 운영체제 모델에 대한 탐색이 진행 중입니다.
