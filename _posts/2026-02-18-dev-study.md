## 핵심 요약

데이터베이스는 데이터를 체계적으로 저장하고 효율적으로 관리하기 위한 시스템입니다. 현대 사회의 모든 정보 시스템은 데이터베이스를 기반으로 운영되며, 이는 응용 프로그램의 핵심 구성 요소입니다. 데이터를 효과적으로 저장하고 검색하며, 여러 사용자가 동시에 안전하게 접근할 수 있도록 보장하는 것이 중요합니다. 관계형 데이터베이스와 에스큐엘이 전통적인 주류였지만, 빅데이터와 분산 시스템의 등장으로 노에스큐엘 데이터베이스도 널리 사용되고 있습니다. 이 학습 자료를 통해 데이터베이스의 기본 개념부터 최신 동향까지 폭넓게 이해할 수 있을 것입니다. 컴퓨터공학을 전공하는 여러분에게 데이터베이스는 필수적인 지식 분야입니다.

## 데이터베이스란?

- 데이터베이스는 구조화된 데이터를 저장, 관리, 검색하는 시스템입니다.
- 다수의 사용자가 동시에 접근하여 데이터를 공유하고 활용할 수 있도록 돕습니다.
- 데이터의 중복을 최소화하고 일관성과 무결성을 유지하는 것이 핵심 목표입니다.
- 응용 프로그램이 데이터를 저장하고 조회하는 데 필요한 기반을 제공합니다.
- 데이터베이스 관리 시스템(디비엠에스)은 이러한 데이터베이스를 제어하고 운영하는 소프트웨어입니다.
- 파일 시스템의 한계를 극복하고 대규모 데이터 처리를 가능하게 합니다.
- 모든 정보 시스템의 백엔드에서 필수적인 역할을 수행합니다.

## 관계형 데이터베이스

- 가장 널리 사용되는 데이터베이스 유형으로, 데이터를 테이블 형태로 구성합니다.
- 테이블은 행(레코드)과 열(속성)로 이루어져 있으며, 서로 관계를 가집니다.
- 고유한 식별자인 기본 키를 통해 각 행을 구별합니다.
- 외래 키를 사용하여 테이블 간의 관계를 정의하고 데이터 무결성을 유지합니다.
- 정규화 과정을 통해 데이터 중복을 줄이고 이상 현상을 방지합니다.
- 강력한 데이터 일관성과 트랜잭션 처리를 보장하는 장점이 있습니다.
- 오라클, 마이에스큐엘, 포스트그레스큐엘, 에스큐엘 서버 등이 대표적인 관계형 데이터베이스 관리 시스템입니다.

## 에스큐엘: 데이터베이스 언어

- 구조화 질의어는 관계형 데이터베이스와 상호작용하는 표준 언어입니다.
- 데이터 정의어(디디엘)를 사용하여 테이블, 뷰, 인덱스 등 데이터베이스 객체를 생성, 수정, 삭제합니다.
- 데이터 조작어(디엠엘)를 통해 데이터를 조회, 삽입, 갱신, 삭제합니다.
- 데이터 제어어(디씨엘)는 데이터에 대한 접근 권한을 부여하거나 회수하는 데 사용됩니다.
- 선언적인 특징을 가지며, 사용자가 원하는 결과를 명시하면 시스템이 최적의 방법을 찾아 수행합니다.
- 복잡한 질의를 작성하여 여러 테이블의 데이터를 결합하고 집계할 수 있습니다.
- 데이터베이스 관리자 및 개발자 모두에게 필수적인 역량입니다.

## 데이터베이스 설계

- 데이터베이스 설계는 시스템 요구사항을 분석하여 효율적인 데이터 구조를 만드는 과정입니다.
- 개념적 설계는 실세계의 엔티티와 관계를 파악하여 엔티티 관계 다이어그램으로 표현합니다.
- 논리적 설계는 엔티티 관계 다이어그램을 관계형 모델의 테이블, 속성, 관계로 매핑하는 단계입니다.
- 물리적 설계는 실제 데이터베이스 관리 시스템에 맞는 저장 구조와 인덱스를 결정합니다.
- 정규화는 데이터 중복을 최소화하고 데이터 이상 현상을 방지하는 중요한 설계 원칙입니다.
- 성능, 확장성, 무결성, 보안 등 다양한 요소를 고려하여 최적의 설계를 목표로 합니다.
- 잘 설계된 데이터베이스는 시스템의 안정성과 유지보수성을 크게 향상시킵니다.

## 트랜잭션과 무결성

- 트랜잭션은 데이터베이스에서 수행되는 하나 이상의 연산을 묶는 논리적인 작업 단위입니다.
- 아시드 속성(원자성, 일관성, 고립성, 지속성)을 만족하여 데이터의 신뢰성을 보장합니다.
- 원자성은 트랜잭션 내의 모든 작업이 성공하거나 모두 실패하여 롤백되는 것을 의미합니다.
- 일관성은 트랜잭션 실행 후에도 데이터베이스의 일관성 제약 조건이 유지됨을 보장합니다.
- 고립성은 여러 트랜잭션이 동시에 실행될 때 서로에게 영향을 주지 않음을 뜻합니다.
- 지속성은 성공적으로 커밋된 트랜잭션의 결과가 영구적으로 반영됨을 의미합니다.
- 동시성 제어는 여러 트랜잭션이 충돌 없이 동시에 실행되도록 관리하는 기술입니다.
- 데이터 무결성은 데이터의 정확성, 유효성, 일관성을 유지하는 것입니다.

## 노에스큐엘 데이터베이스

- 전통적인 관계형 데이터베이스의 한계를 극복하기 위해 등장한 비관계형 데이터베이스입니다.
- 빅데이터 처리, 고가용성, 수평적 확장성에 유리하며 유연한 데이터 모델을 가집니다.
- 키-값 저장소, 문서 데이터베이스, 열 지향 데이터베이스, 그래프 데이터베이스 등 다양한 유형이 있습니다.
- 관계형 데이터베이스처럼 스키마가 고정되지 않아 데이터 구조 변경에 유연하게 대응합니다.
- 일관성보다는 가용성과 분할 허용성을 중시하는 캡 이론에 기반합니다.
- 몽고디비, 카산드라, 레디스, 네오포제이 등이 대표적인 노에스큐엘 데이터베이스입니다.
- 로그 데이터, 실시간 분석, 콘텐츠 관리 시스템 등 특정 워크로드에 특히 효과적입니다.
- 다양한 데이터 형식과 대량의 비정형 데이터를 효율적으로 처리할 수 있습니다.

## 최신 흐름

- 클라우드 데이터베이스의 확산: 아마존 아우로라, 구글 클라우드 스패너 등 클라우드 기반 데이터베이스 서비스가 대세입니다.
- 분산 데이터베이스 시스템의 발전: 샤딩, 복제 기술을 통해 대규모 데이터를 여러 서버에 분산하여 처리합니다.
- 인메모리 데이터베이스의 성능 향상: 데이터를 주 기억 장치에 저장하여 극도로 빠른 데이터 처리 속도를 제공합니다.
- 그래프 데이터베이스의 부상: 복잡한 관계 데이터를 효율적으로 모델링하고 분석하는 데 활용됩니다.
- 시계열 데이터베이스의 전문화: 센서 데이터, 로그 데이터 등 시간 순서에 따라 발생하는 데이터를 저장하고 분석하는 데 최적화되었습니다.
- 데이터 패브릭 및 메시 아키텍처: 분산된 이종 데이터 소스를 통합하고 관리하는 새로운 패러다임이 주목받고 있습니다.
- 인공지능 및 머신러닝 통합: 데이터베이스 내에서 직접 머신러닝 모델을 훈련하고 추론하는 기능이 강화되고 있습니다.
- 서버리스 데이터베이스: 인프라 관리 없이 데이터베이스 기능을 사용할 수 있어 개발 편의성이 높아지고 있습니다.
- 데이터 거버넌스 및 프라이버시 강화: 데이터 보안 및 규제 준수의 중요성이 커지며 관련 기술과 솔루션이 발전하고 있습니다.
- 블록체인 기반 분산 원장 기술(디엘티)과의 연계: 투명하고 불변하는 데이터 저장 방식이 특정 산업에서 활용 가능성을 모색 중입니다.
