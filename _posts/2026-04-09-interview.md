## 오늘의 면접 주제
> React와 JavaScript 학습 콘텐츠가 다시 주목받으면서, 클로저는 프론트엔드와 Node.js 면접에서 꾸준히 나오는 핵심 주제입니다.

## 핵심 한 줄 요약
클로저는 함수가 선언될 때의 스코프를 기억해서, 함수 바깥 변수를 나중에도 계속 사용할 수 있게 하는 동작입니다.

## 면접 질문 & 모범 답변

### Q1. 클로저란 무엇인가요?
**기본 답변**: 클로저는 함수와 그 함수가 선언될 당시의 렉시컬 환경을 함께 묶어서 이해하는 개념입니다. 그래서 함수가 호출되는 위치와 상관없이, 자신이 원래 보던 변수에 계속 접근할 수 있습니다.  
**심화 포인트**: 많은 지원자가 "함수가 변수를 기억한다"까지만 말합니다. 더 좋은 답변은 "실행 시점이 아니라 선언 시점의 스코프를 따른다"까지 설명하는 것입니다.

### Q2. 렉시컬 스코프와 클로저는 어떤 관계인가요?
**기본 답변**: 렉시컬 스코프는 변수의 유효 범위가 코드가 작성된 위치로 결정된다는 뜻입니다. 클로저는 이 렉시컬 스코프 덕분에 동작합니다.  
**심화 포인트**: 호출한 위치가 아니라 선언한 위치가 기준입니다. 이 차이를 모르면 실행 컨텍스트와 스코프를 섞어서 설명하게 됩니다.

### Q3. 클로저는 왜 필요한가요?
**기본 답변**: 상태를 안전하게 감추고, 필요한 함수만 외부에 노출할 수 있기 때문입니다. 일종의 private 변수처럼 사용할 수 있습니다.  
**심화 포인트**: 객체 없이도 데이터를 캡슐화할 수 있다는 점이 중요합니다. 면접에서는 "전역 변수 오염을 줄인다"는 실무적 이유를 같이 말하면 좋습니다.

```js
function counter() {
  let count = 0;
  return () => ++count;
}
```

### Q4. 실무에서 클로저를 어떻게 활용했나요?
**기본 답변**: 이벤트 핸들러, debounce, throttle, factory 함수에서 자주 사용합니다. 특히 React에서도 handler가 바깥 상태를 참조할 때 클로저가 자연스럽게 쓰입니다.  
**심화 포인트**: 단순히 "많이 쓴다"가 아니라, 어떤 값을 고정해서 재사용하려고 클로저를 쓴다고 설명해야 합니다. 예를 들어 API base URL이나 설정값을 묶는 경우가 있습니다.

### Q5. 커링과 메모이제이션에서 클로저는 어떻게 쓰이나요?
**기본 답변**: 커링은 일부 인자를 먼저 받아 두고, 나중에 나머지 인자를 받는 방식입니다. 메모이제이션은 이전 계산 결과를 저장해 재사용하는 방식입니다. 둘 다 바깥 함수의 변수를 내부 함수가 계속 참조하므로 클로저를 사용합니다.  
**심화 포인트**: 커링은 함수 재사용성과 조합성을 높입니다. 메모이제이션은 성능 최적화에 도움되지만, 캐시가 계속 쌓이면 메모리 사용량이 늘 수 있습니다.

### Q6. 클로저가 메모리 누수를 만들 수도 있나요?
**기본 답변**: 네, 필요 없는 데이터가 클로저에 계속 참조되면 GC 대상이 되지 않을 수 있습니다. 특히 큰 객체나 DOM 참조를 오래 잡고 있으면 문제가 됩니다.  
**심화 포인트**: 클로저 자체가 누수를 만드는 것은 아닙니다. "더 이상 필요 없는 참조를 오래 유지하는 코드"가 문제입니다. 원인을 클로저에만 돌리면 설명이 얕아 보입니다.

### Q7. 클로저와 전역 변수 사용의 차이는 무엇인가요?
**기본 답변**: 전역 변수는 어디서든 접근 가능해서 예측이 어렵습니다. 클로저는 필요한 함수만 상태에 접근하게 제한할 수 있습니다.  
**심화 포인트**: 면접에서는 "접근 가능 범위를 줄여 변경 지점을 통제한다"는 표현이 좋습니다. 유지보수성과 테스트 용이성으로 연결하면 더 좋습니다.

### Q8. 면접에서 클로저를 설명할 때 가장 중요한 포인트는 무엇인가요?
**기본 답변**: 정의만 말하지 말고, 선언 시점의 스코프를 기억한다는 점과 실제 활용 사례를 함께 말해야 합니다. 그리고 장점과 주의점까지 균형 있게 설명하는 것이 중요합니다.  
**심화 포인트**: 좋은 답변은 개념, 동작 원리, 활용, 위험 요소를 한 흐름으로 묶습니다. 암기한 정의만 말하면 실무 이해도가 낮아 보일 수 있습니다.

## 헷갈리기 쉬운 포인트
클로저는 "특별한 함수 종류"가 아닙니다. 함수가 외부 스코프를 참조하는 일반적인 동작을 설명하는 개념입니다.
클로저는 호출 시점의 변수를 보는 것이 아닙니다. 선언 시점의 스코프를 따릅니다.
메모리 누수는 클로저 때문이 아니라, 불필요한 참조를 계속 유지한 코드 때문에 생깁니다.

## 면접관 시각
면접관은 지원자가 정의를 외웠는지보다, 왜 그런 동작이 생기는지 이해했는지를 봅니다. 렉시컬 스코프와 연결해서 설명하고, 커링이나 메모이제이션 같은 실용 예시를 말할 수 있으면 좋습니다. 여기에 메모리 관리 관점까지 짚으면, 단순 문법 암기를 넘어 실제로 JavaScript를 써 본 사람으로 보입니다.
